# Draft: Add logger pattern to all callAgent call sites

**開発種別:** 機能追加（既存基盤の有効化）

**目的:** spec 151 で整備済みの AgentLog + logger 基盤を、全 callAgent/callAgentAsync 呼び出し元（19 箇所）で有効化し、エージェント呼び出しのログ記録を実現する。

## Q&A

### Q1: スコープ — flow 系と docs 系を分けるか？
全 19 箇所を一括で対応する。実装パターンが共通で、分割するメリットが薄い。

### Q2: sync 関数内での async logger の扱い
fire-and-forget（`logger().catch(() => {})`）で呼ぶ。ログ欠損のリスクは許容する。
対象: run-gate.js (2), get-context.js (1), init.js (1), template-merger.js (1) の計 5 箇所。
将来的に loggerSync を追加する構想あり（ボード ffe7）。

### Q3: Logger のインスタンス化方針
シングルトン方式を採用する。CLI エントリポイントで初期化し、各呼び出し元はシングルトン経由で書き込む。
- 初期化されていない場合: console.error で警告を出し、ログ書き込みをスキップする。エラーは投げない。
- cfg はシングルトンが保持するため、各呼び出し元で渡す必要なし。

### Q4: ログ記録の手間を最小化する
- 呼び出し側は「ログ生成 → エージェント呼び出し → ログ書き込み」の 3 ステップのみ意識する
- 開始時刻・終了時刻・経過時間の計測はログ基盤が自動で行い、呼び出し側に計測コードを書かせない

### Q5: 例外時のログ記録
- エージェント呼び出しが例外で終了した場合: ログを記録する（実行時間が記録できて障害調査に有用）
- エージェント呼び出しが正常終了した場合: ログを記録する

## 設計決定サマリー

| 項目 | 決定 |
|------|------|
| スコープ | 全 19 箇所一括 |
| sync 呼び出し元の logger | fire-and-forget |
| Logger 方式 | シングルトン + sdd-forge.js で init |
| 未初期化時 | console.error 警告 + スキップ |
| ログ記録の手間 | 呼び出し側は 3 ステップのみ、計測は基盤が自動化 |
| 例外時 | try/finally でログを残す |

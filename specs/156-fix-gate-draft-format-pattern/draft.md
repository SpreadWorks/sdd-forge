# Draft: gate draft のフォーマット検出パターン緩和

**開発種別:** バグ修正

**目的:** gate draft のフォーマット検出で `## 見出し形式` を受け付けるよう正規表現を拡張し、コロン形式と見出し形式の両方を受け入れる。

---

## Q&A

**Q1: 現状どのパターンが受け付けられていて、何が拒否されているか？**

A: `src/flow/lib/run-gate.js` の `checkDraftText()` 関数（L136–L146）にて確認済み。

現状の正規表現:
- 開発種別: `/\*{0,2}(?:開発種別|dev(?:elopment)?\s*type)\*{0,2}\s*[:：]/i`
- 目的: `/\*{0,2}(?:目的|goal)\*{0,2}\s*[:：]/i`

受け付けているもの:
- `**開発種別:** 機能追加` ← PASS
- `開発種別: 機能追加` ← PASS

受け付けていないもの:
- `## 開発種別` ← FAIL（現状の正規表現にマッチしない）
- `## 目的` ← FAIL（同上）

**Q2: 改善の要件は何か？**

A: `## 見出し形式` と `key: 形式（コロン形式）` の両方を受け付けるよう正規表現を拡張する。
既存のPASS条件を壊さない。日本語・英語両対応も維持する。

**Q3: スコープの境界は？**

A: 対象は `checkDraftText()` 内の `hasDevType` と `hasGoal` の2つの正規表現のみ。
他のフェーズ（spec/impl）の正規表現は変更しない。

**Q4: テストへの影響は？**

A: `src/flow/lib/run-gate.js` から `checkDraftText` がエクスポートされているため、
既存のテストを確認してカバレッジを評価する必要がある。

**Q5: エッジケースは？**

A: `## 開発種別` の後に何も書かれていないケース（見出し行だけ）も PASS とする。
要件上、見出しがあれば記入されているとみなすため。

**Q6: spec/impl フェーズの検出項目に同様の問題はないか？**

A: `checkSpecText()` を確認した（同ファイル L45–L110）。
spec フェーズの必須セクション（## Clarifications、## Open Questions、## User Confirmation 等）は
`/^\s*##\s+SectionName/im` という見出し形式で検出されている。
spec フェーズは元々見出し形式を使っているため、コロン形式 vs 見出し形式の問題は発生しない。

つまり今回の問題は `checkDraftText()` に固有のもの。他フェーズへの波及はない。

# Feature Specification: 087-fix-acceptance-test-fixtures

**Feature Branch**: `feature/087-fix-acceptance-test-fixtures`
**Created**: 2026-03-22
**Status**: Draft
**Input**: GitHub Issue #12

## Goal

Fix acceptance test fixture configuration errors, add missing source files, and rename shells ‚Üí commands across webapp preset chain to unify console command handling for all frameworks.

## Scope

### A. Fixture config type bugs
- `tests/acceptance/fixtures/php/.sdd-forge/config.json`: change `type: "php"` ‚Üí `"php-webapp"`
- `tests/acceptance/fixtures/node/.sdd-forge/config.json`: change `type: "node"` ‚Üí `"js-webapp"`

### B. shells ‚Üí commands rename (webapp preset chain)
- Rename `src/presets/webapp/data/shells.js` ‚Üí `commands.js` (class: `ShellsSource` ‚Üí `CommandsSource`)
- Rename `src/presets/cakephp2/data/shells.js` ‚Üí `commands.js` (class: `CakephpShellsSource` ‚Üí `CakephpCommandsSource`)
- New `src/presets/laravel/data/commands.js` ‚Äî match `app/Console/Commands/*.php`
- New `src/presets/symfony/data/commands.js` ‚Äî match `src/Command/*.php`
- Rename template `batch_and_shell.md` ‚Üí `commands.md` (en + ja)
- Update `webapp/preset.json` chapters: `batch_and_shell.md` ‚Üí `commands.md`
- Update all `{{data}}` directives: `webapp.shells.list` ‚Üí `webapp.commands.list`
- Update CakePHP2 template overrides: `cakephp2.shells.*` ‚Üí `cakephp2.commands.*`
- Update analysis key: `analysis.shells` ‚Üí `analysis.commands`
- Update all references in tests

### C. CakePHP2 fixture missing files
Add stub files to `tests/acceptance/fixtures/cakephp2/`:
- `app/Lib/` ‚Äî library/behavior stubs
- `app/View/Helper/` ‚Äî helper stubs
- `app/View/Layouts/` ‚Äî layout stubs
- `app/View/Elements/` ‚Äî element stubs
- `app/Console/Command/` ‚Äî shell command stubs
- `docker-compose.yml`
- ACL config, email templates as needed

### D. Laravel fixture missing files
Add stub files to `tests/acceptance/fixtures/laravel/`:
- `app/Http/Middleware/` ‚Äî middleware stubs
- `app/Console/Commands/` ‚Äî Artisan command stubs

## Backward Compatibility

The `analysis.shells` key is renamed to `analysis.commands`. Per the project's alpha-version policy, backward-compatible code is not written. Old analysis.json files are regenerated by running `sdd-forge scan`. No migration shim is needed.

## Out of Scope

- Adding new chapters or templates beyond the rename
- Changing scan logic beyond match() overrides for commands
- Symfony fixture missing files (not mentioned in Issue #12)
- Performance optimization of acceptance tests

## Requirements

1. **[P1] Config type fix**: When acceptance tests resolve presets for php and node fixtures, they shall find valid presets. Change php fixture to `type: "php-webapp"` and node fixture to `type: "js-webapp"`
2. **[P1] Rename shells ‚Üí commands**: All source files, classes, analysis keys, template directives, and test references updated
3. **[P1] Laravel commands DataSource**: `match()` detects `app/Console/Commands/*.php` (excluding base Command class)
4. **[P1] Symfony commands DataSource**: `match()` detects `src/Command/*.php` (excluding abstract base)
5. **[P2] CakePHP2 fixture files**: When acceptance tests run the CakePHP2 fixture through the scan‚Üídata pipeline, DataSources that read Lib/View/Console categories shall find matching files. Add the following stubs (each containing a minimal valid PHP class):
   - `app/Lib/AppLib.php` ‚Äî PHP class with `class AppLib {}` and at least one public method
   - `app/View/Helper/AppHelper.php` ‚Äî PHP class extending `AppHelper` with one method
   - `app/View/Layouts/default.ctp` ‚Äî CakePHP layout template with `<?php echo $content_for_layout; ?>`
   - `app/View/Elements/sidebar.ctp` ‚Äî CakePHP element with basic HTML structure
   - `app/Console/Command/CleanupShell.php` ‚Äî PHP class named `CleanupShell` with a `main()` method
   - `docker-compose.yml` ‚Äî YAML with at least `services.app` and `services.db` keys
6. **[P2] Laravel fixture files**: When acceptance tests run the Laravel fixture through the scan‚Üídata pipeline, the commands DataSource and middleware detection shall find matching files. Add the following stubs:
   - `app/Http/Middleware/Authenticate.php` ‚Äî PHP class `Authenticate` extending `Middleware` with `handle()` method
   - `app/Console/Commands/CleanupCommand.php` ‚Äî PHP class `CleanupCommand` extending `Command` with `$signature` property and `handle()` method
7. **[P1] Template rename**: When `sdd-forge init` copies webapp chapter templates, the commands chapter shall be named `commands.md` (not `batch_and_shell.md`). Rename template files in webapp preset (en + ja) and update `preset.json` chapters array accordingly

## Acceptance Criteria

- `npm test` passes with no regressions
- Acceptance tests for php and node fixtures run without preset resolution errors
- `{{data("webapp.commands.list")}}` resolves correctly for CakePHP2, Laravel, and Symfony fixtures
- No remaining references to `shells` in source code (except git history)
- CakePHP2 fixture has files for all major app directories
- Laravel fixture has middleware and Artisan command stubs

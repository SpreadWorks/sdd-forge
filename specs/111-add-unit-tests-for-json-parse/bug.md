# テスト中に発見したバグ

## 背景

spec 111 の auto mode で fixUnescapedQuotes のテストを書いた際、2つのテストケースでプロダクトコードのバグを発見した。spec には「プロダクトコード変更なし」とあるためテストを現在の動作に合わせたが、発見したバグがボードに登録されなかった。

## 問題

人間が見ていない auto mode では、テスト失敗 → テスト修正で終わってしまい、発見した問題が記録されずに見落とされる。

## 改善案

- SKILL.md に「テストがプロダクトコードのバグで失敗した場合、テストを修正するだけでなく、発見したバグを redolog に記録せよ」と指示を追加
- または spec.md の Open Questions に「テスト中に発見した問題」セクションを追加する規約

# Draft: improve-translate-prompt

## 概要

`sdd-forge docs translate` の翻訳プロンプトを改善し、直訳的でない自然な翻訳を生成する。

## 現状の問題点

1. **カタカナ過多** — 日本語の適切な語彙がある箇所もカタカナ直訳（overhead → オーバーヘッド、context → コンテキスト）
2. **英語の文構造をそのまま置換** — 主語が長い名詞節になる英語構文がそのまま日本語に
3. **翻訳調の定型表現** — 「本ツール」「以下のテーブルは〜を定義しています」
4. **文体の指示なし** — config.json の `documentStyle.tone` が翻訳に反映されていない

## 決定事項

- **文体**: `documentStyle.tone` に基づいてマッピング
  - `polite` → です/ます調
  - `formal` → である調
  - `casual` → 口語的
- **カタカナ語**: 細かいルールは指定しないが、「直訳的なカタカナ語を避け自然な日本語表現を優先する」旨をプロンプトに含める
- **文構造**: 英語の文構造をそのまま日本語にしない指示を追加
- **冗長表現**: 「〜すること」「〜される」の連続を避ける指示を追加
- **文化考慮**: 対象言語の慣習・文化を考慮して翻訳する指示を追加
- **品質チェック**: 改善前後で同じファイルを翻訳し、品質を比較する

## 対象ファイル

- `src/docs/commands/translate.js` の `translateDocument()` 内の system prompt

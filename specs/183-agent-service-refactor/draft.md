# Draft: Agent サービス化（1877/Phase2A）

**開発種別:** ENHANCE（リファクタリング）

**目的:** AI エージェント呼び出し基盤（現状 700 行超の関数群が散在）を、Container 配下の単一サービスに再設計する。呼び出し側引数の縮小、作業ディレクトリ受け渡しの自動化、同期呼び出し経路の廃止、argv 閾値の設定化を併せて実施し、provider 別のドメイン知識を集約可能な構造に整える。

## 背景

- AI エージェント呼び出し基盤は CLI 全体の中核機能だが、関数群と provider 別データが平坦に並列展開されており、新 provider 追加・呼び出し規約変更時のコストが高い。
- 同期呼び出し経路と非同期呼び出し経路で子プロセス起動方式が分かれており、関連ボード（hang リスク）として指摘済み。
- 一部 provider 設定に作業ディレクトリ値がハードコードされ、設定駆動の作業ディレクトリと整合していない。
- 当該基盤は npm パッケージの公開 API ではなく内部 API のみで、後方互換シムを残す動機はない（alpha 期）。

## 要件

優先順位: **P0**=本 spec の必達、**P1**=同 spec 内で実施するが受け入れ基準は P0 を最優先。

### P0（必達）

- **R1:** When 呼び出し元コードが AI エージェントを起動するとき、shall 共有のエージェントサービス経由でのみ呼び出すこと。直接呼び出し用の旧公開関数を残してはならない。
- **R2:** When エージェント呼び出し API が呼ばれるとき、shall タイムアウトと作業ディレクトリは設定およびプロジェクトコンテキストから内部解決すること。呼び出し元はそれらを引数として渡せない。
- **R3:** When エージェント子プロセスを起動するとき、shall stdin EOF 待ちで hang しない呼び出し方式に統一すること。同期呼び出し経路を残してはならない。
- **R4:** If 特定の AI CLI が作業ディレクトリ受け渡し用 CLI フラグを必要とするとき、shall サービスが設定解決済みの作業ディレクトリを当該フラグ値として自動付与すること。CLI 設定側に作業ディレクトリのハードコード値を残してはならない。
- **R5:** When 設定で argv サイズ閾値が指定されているとき、shall その値を stdin フォールバック判定に使用すること。未指定時は 100000 bytes を既定値とする。
- **R6:** When 出力パース・provider 判定・builtin profile・JSON 出力フラグ・systemPrompt 受け渡し方法のいずれかを参照するとき、shall provider 別の単一定義箇所から取得すること。provider 横断のフラット辞書および command 文字列マッチによる provider 判定を残してはならない。

### P1（同 spec 内実施）

- **R7:** When `npm test` が実行されるとき、shall 既存テスト全件が pass し、エージェントサービスおよび provider 抽象の単体テストが追加されていること。
- **R8:** When 呼び出し元が AI レスポンスを必要とするとき、shall 非同期呼び出しに統一されていること。

## 移行方針

alpha 版ポリシー「後方互換コードは書かない」（CLAUDE.md）に従い、本 spec 内で全呼び出し元を新サービス経由へ書き換え、旧公開関数を削除する。互換シムを残さない。

## 影響

- **既存機能:** docs パイプラインおよび flow 系コマンドが新サービス経由に移行。利用者から見た挙動は不変。
- **設定:** stdin フォールバック閾値の新規キーが追加される（省略時の既定値あり）。既存の AI 関連設定キーは据え置き。
- **テスト:** 既存ユニットテストを新サービスに追従。provider 抽象とサービスの単体テストを追加。
- **CLI コマンド/オプション:** 変更なし（影響は内部 API のみ）。

## 制約・ガードレール

- 外部依存追加禁止（Node.js 組み込みのみ）。CLAUDE.md「外部依存なし」。
- alpha 版ポリシー: 後方互換コードを書かない。旧 API は削除。CLAUDE.md「alpha 版ポリシー」。
- `src/` 配下にプロジェクト固有情報を含めない。CLAUDE.md「`src/` の禁止事項」。
- OOP による型表現: 意味のある値・サービスは、コンストラクタで invariant を強制するクラスとして定義する（CLAUDE.md「OOP による型表現」）。
- AI エージェントの非同期呼び出しは子プロセスの stdin EOF 待ちで hang しない方式を必須とする。

### Q3. 既存コード置換方針
**A:** 本 spec 内で全呼び出し元を新サービス経由に書き換え、旧公開関数を削除する（deprecated 残し無し）。
**根拠:** CLAUDE.md「alpha 版ポリシー: 後方互換コードは書かない」。当該基盤は npm パッケージの公開 API ではなく呼び出し元は内部のみ（事前の grep 確認）。

### Q4. サービス公開の方向性
**A:** AI エージェント呼び出しを共有サービスとして公開し、呼び出し元はサービス経由でのみアクセスする方針。直接 import の関数呼び出しは廃止する。
**根拠:** Issue 指摘 17-4、CLAUDE.md「シンプルなインターフェースに十分な実装を隠す」。

### Q5. Provider 責務集約の方向性
**A:** provider 別の出力パース・JSON 出力フラグ・systemPrompt 渡し方・workDir フラグ・builtin profile 一覧を、provider ごとに 1 箇所にまとめる方針。新 provider 追加が単一定義の追加で済むことを目標とする。
**根拠:** Issue 指摘 22「parse/usage/capability の凝集」、CLAUDE.md「OOP による型表現」「Open/Closed」。現状これらは複数箇所に散在し、新 provider 追加時に複数箇所の変更を要する。

### Q6. 作業ディレクトリ受け渡しの方向性
**A:** provider 自身が作業ディレクトリ受け渡し用 CLI フラグを宣言し、サービス側が設定解決済みの作業ディレクトリを自動付与する方針。provider 設定側に作業ディレクトリ値を直接書かない。
**根拠:** Issue 指摘 19、設定で解決される作業ディレクトリを単一情報源とする既存設計と整合。

### Q7. argv 閾値の設定化 + 同期/非同期統一
**A:** (a) argv サイズ閾値を設定キーとして外部化（既定 100000）。(b) 同期呼び出し経路を廃止し、非同期呼び出しに統一する。
**根拠:** (a) Issue 指摘 21「命名修正 + config 化」。(b) Issue 指摘 16 + 関連ボード 227f「同期/非同期で子プロセス起動方式が分かれており hang リスク」。

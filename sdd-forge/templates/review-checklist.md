# temp/docs レビュー基準

## 目的
- テンプレート準拠だけでなく、設計判断に必要な情報密度を担保する。

## チェック項目

1. 構成
- 全ファイルに `## 説明` と `## 内容` がある。
- 01〜10 の10ファイルが存在する。
- `docs/*.md` に `AUTO-GEN` と `MANUAL` の両ブロックがある。

2. 環境差分
- local/staging/production の比較表がある（該当章）。

3. 認証/セッション
- `05` に AuthComponent と ACL の責務が明記される。
- `05` にセッション管理の方式が記載される。

4. 運用
- `04` に Docker セットアップ・npm scripts（Docker 操作）がある。
- `02` にデプロイフローがある。

5. DB
- `06` にマルチDB構成と切り替え方式がある。
- `07` にテーブル整理と主要 FK 関係がある。

6. コントローラ
- `08` にコントローラ一覧とルーティング設定がある。
- `08` に主要アクションが記載される。

7. ビジネスロジック
- `09` に Model/Logic 一覧がある。
- `09` に JASRAC/Crankin/Hollywood 連携が記載される。

8. バッチ
- `10` に Shell コマンド一覧がある。
- `10` に ViewLog/ViewReport バッチが記載される。

9. 具体性
- あいまい語だけで終わらず、クラス名・テーブル名・設定ファイル名が明記される。

10. 手動セクション境界
- `## 開発・運用Tips` は自動生成ソースに含めない。
- Tips は `MANUAL` ブロック内でのみ運用する。

## 合格条件
- 重大欠落（コントローラ群不足、DB構造不足、バッチ未記載）が 0 件。

### 汎用フォントの直接使用禁止
<!-- {%meta: {phase: [impl]}%} -->
汎用フォント名（Arial, Times New Roman, Helvetica 等）を直接使用してはならない。プロジェクトのフォントを CSS カスタムプロパティまたはデザイントークンとして定義し、それを参照すること。

### カラーシステムの使用必須
<!-- {%meta: {phase: [impl]}%} -->
すべての色は CSS カスタムプロパティ（例: `--color-primary`）で定義すること。コンポーネントスタイル内でのハードコードされた hex/rgb 値を禁止する。プロジェクトのカラーシステムを使用すること。

### アニメーション手法の制約
<!-- {%meta: {phase: [impl]}%} -->
JavaScript によるアニメーションよりも CSS アニメーション・トランジションを優先すること。CSS で実現できない効果（スクロール連動、物理演算等）の場合のみ JavaScript を使用すること。

### コンポーネントでのインラインスタイル禁止
<!-- {%meta: {phase: [impl]}%} -->
コンポーネントテンプレート内の `style` 属性を禁止する。CSS クラス、CSS Modules、styled-components を使用すること。

### レスポンシブブレークポイントはデザイントークンを使用
<!-- {%meta: {phase: [impl]}%} -->
メディアクエリのブレークポイントはデザイントークンまたは共有変数を参照すること。個別コンポーネントでのハードコードされたピクセル値を禁止する。

### インタラクティブ要素のアクセシビリティ最低基準
<!-- {%meta: {phase: [spec, impl]}%} -->
すべてのインタラクティブ要素（ボタン、リンク、フォームコントロール）にはアクセシブルなラベル、十分なカラーコントラスト（WCAG AA）、キーボード操作性を確保すること。

### 画像フォーマットと最適化の必須化
<!-- {%meta: {phase: [impl]}%} -->
画像はモダンフォーマット（WebP, AVIF）を使用し、適切なフォールバックを提供すること。未最適化の画像（BMP, 非圧縮 PNG/TIFF）のプロダクションビルドへの含有を禁止する。

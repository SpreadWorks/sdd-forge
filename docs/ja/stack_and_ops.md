# 技術スタックと運用

<!-- {{data("monorepo.monorepo.apps", {labels: "stack_and_ops", ignoreError: true})}} -->
<!-- {{/data}} -->

## 説明

<!-- {{text({prompt: "この章の概要を1〜2文で記述してください。使用言語・フレームワーク・主要ツールのバージョンを踏まえること。"})}} -->

この章で確認できる運用関連の実装は、CakePHP 2.x 向け Docker 設定の拡張ポイントと、GitHub Actions のワークフロー YAML を解析する CI/CD パイプライン用 DataSource です。
主要な対象は CakePHP 2.x、Docker、GitHub Actions であり、CI/CD 側ではワークフロー名、トリガー、ジョブ、シークレット、環境変数を抽出できる実装があります。
<!-- {{/text}} -->

## 内容

### 技術スタック

| カテゴリ | 技術名 | バージョン |
| --- | --- | --- |
| Web フレームワーク | CakePHP | 2.x |
| コンテナ関連 | Docker | 解析データに記載なし |

### 依存パッケージ

<!-- {{text({prompt: "プロジェクトの依存パッケージ管理方法を説明してください。"})}} -->

提供された解析データには、アプリケーションの依存パッケージを管理する仕組み自体は含まれていません。
確認できるのは、CI/CD 用の `PipelinesSource` が GitHub Actions のワークフロー YAML を解析し、各ジョブの `uses` 依存を抽出して重複排除する実装です。
このため、本章の範囲で説明できる依存管理は、ワークフロー内で参照される GitHub Actions 依存の可視化までです。
アプリケーション本体のパッケージ管理ツールや運用手順については、解析データからは確認できません。
<!-- {{/text}} -->

### デプロイフロー

<!-- {{text({prompt: "デプロイの手順とフローを説明してください。"})}} -->

デプロイに直接対応する専用フローは、提供された解析データからは確認できません。
運用面で実装されているのは、`.github/workflows/*.yml` を対象に GitHub Actions のワークフローを解析し、パイプライン一覧、ジョブ詳細、シークレット、環境変数を表形式で出力する機能です。
この解析では、`on:` セクションからトリガー、ブランチ条件、cron スケジュールを取得し、`jobs:` セクションからジョブ ID、`runs-on`、ステップ数、`uses` 依存を抽出できます。
そのため、デプロイフローを確認する際は、対象ワークフローに定義されたトリガー条件、実行ジョブ、必要なシークレットと環境変数をこの DataSource で整理する運用になります。
<!-- {{/text}} -->

### 運用フロー

<!-- {{text({prompt: "運用手順を説明してください。"})}} -->

運用手順として確認できるのは、CI/CD パイプライン定義と Docker 設定の扱いです。
CI/CD では `PipelinesSource` を利用して GitHub Actions のワークフロー YAML を解析し、パイプライン一覧は `list()`、ジョブ詳細は `jobs()`、シークレットと環境変数は `env()` で確認できます。
これにより、どのイベントでワークフローが起動するか、どのブランチが対象か、各ジョブがどの実行環境で何ステップ動くかを把握できます。
また、`extractRefs()` により `secrets.*` と `env.*` の参照が抽出されるため、運用時に必要な設定値の確認に利用できます。
一方で、CakePHP 2.x 向けの Docker 設定 DataSource は現時点ではスタブ実装で、`list()` は常に `null` を返します。
そのため、Docker 関連の運用情報は拡張ポイントとして用意されていますが、現状の実装では実データの確認には使えません。
<!-- {{/text}} -->

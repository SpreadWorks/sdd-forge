# 02. 技術スタックと運用

## 説明

<!-- {{text: この章の概要を1〜2文で記述してください。使用言語・フレームワーク・主要ツールのバージョンを踏まえること。}} -->

sdd-forge は Node.js >=18.0.0 上で動作する JavaScript（ES Modules）製の CLI ツールであり、外部依存パッケージを一切持ちません。この章では、技術スタック、依存パッケージの管理方針、デプロイ手順および運用フローについて説明します。

### 依存パッケージ

<!-- {{text: プロジェクトの依存パッケージ管理方法を説明してください。}} -->

sdd-forge は**外部依存ゼロ**のポリシーを採用しています。すべての機能は Node.js 組み込みモジュール（`node:fs`、`node:path`、`node:test`、`node:assert` など）のみで実装されています。

`package.json` へのサードパーティパッケージの追加は禁止されています。このポリシーにより、パッケージの軽量性を維持し、サプライチェーンリスクを排除し、依存関係の監査やロックファイルのメンテナンスを不要にしています。

`package.json` の `"files"` フィールドは `["src/"]` に設定されており、公開パッケージには `src/` ディレクトリと `package.json`、`README.md`、`LICENSE` のみが含まれます。

### デプロイフロー

<!-- {{text: デプロイの手順とフローを説明してください。}} -->

sdd-forge は npm パッケージとして配布されます。リリースは手動の 2 ステップ手順で行います。

1. **公開前チェック** — `npm pack --dry-run` を実行し、認証情報や `.env` などの機密ファイルがパッケージに含まれていないことを確認します。
2. **バージョン更新** — `npm version patch`、`npm version minor`、または `package.json` の直接編集でバージョンを更新します。
3. **alpha タグで公開** — `npm publish --tag alpha` を実行し、`alpha` dist-tag でパッケージを公開します。
4. **latest タグへの昇格** — `npm dist-tag add sdd-forge@<version> latest` を実行し、npm レジストリの `latest` タグを更新します。

ステップ 3 と 4 の両方が必須です。`--tag alpha` のみの公開では `latest` タグが更新されず、npmjs.com のパッケージページに新バージョンが反映されません。

自動化された CI/CD パイプラインはありません。公開はメンテナーがリリースの意図を明示した場合にのみ、手動で実行されます。

### 運用フロー

<!-- {{text: 運用手順を説明してください。}} -->

ローカル開発および日常の運用は以下のワークフローに従います。

- **ローカルセットアップ** — リポジトリルートで `npm link` を実行し、`sdd-forge` を開発用のグローバル CLI コマンドとして登録します。
- **テストの実行** — `npm test` を実行します。`tests/` 配下のすべての `*.test.js` ファイルを検出し、Node.js 組み込みテストランナー（`node --test`）で実行します。アサーションには `node:assert/strict` を、テスト補助には `tests/helpers/` のユーティリティを使用します。
- **パッケージ検証** — テストスイートには `tests/package.test.js` が含まれており、`npm pack --dry-run` の出力が `src/` の想定ファイルのみであることを検証し、機密データの混入を防止します。
- **ドキュメント生成** — `sdd-forge docs build` を実行すると、ドキュメント生成パイプライン全体（`scan → enrich → init → data → text → readme → agents → [translate]`）が実行されます。各ステージは個別に実行することも可能です。
- **SDD ワークフロー** — 機能開発は Spec-Driven Development フローに従います。`sdd-forge flow --request "<要望>"` でプランニングを開始し、実装・マージの各ステージへ進みます。

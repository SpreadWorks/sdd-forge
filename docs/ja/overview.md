# ツール概要とアーキテクチャ

## 説明

<!-- {{text({prompt: "この章の概要を1〜2文で記述してください。ツールの目的・解決する課題・主要なユースケースを踏まえること。"})}} -->

このツールは、ソースコード解析に基づくドキュメント生成と、Spec-Driven Development ワークフローを CLI で扱うための基盤を提供します。`docs/` 配下の章構成、言語切替、前後ナビゲーション、プロジェクトメタデータを動的に組み立て、利用者が保守しやすいドキュメントと手順を得られるようにします。
<!-- {{/text}} -->

## 内容

### ツールの目的

<!-- {{text({prompt: "このCLIツールが解決する課題と、ターゲットユーザーを説明してください。ソースコードの package.json や README から目的を読み取ること。"})}} -->

この CLI ツールは、ドキュメントの章一覧、言語切替、前後リンク、プロジェクト名・説明・バージョン・スクリプト一覧といった情報を、ソースコードや設定ファイルから自動で取り出してマークダウンへ反映するための仕組みを提供します。

`docs/` ディレクトリ、`.sdd-forge/config.json`、`package.json`、`locale/<lang>/ui.json` を参照して表示内容を組み立てるため、手作業でナビゲーションやメタ情報を同期する負担を減らせます。

対象となる利用者は、プロジェクトの利用者向けドキュメントを継続的に整備したい開発者や、複数言語のドキュメントや章構成を一貫した形で管理したいチームです。あわせて、Spec-Driven Development の手順を CLI 上で扱いたい利用者にも向いています。
<!-- {{/text}} -->

### 主要コンセプト

<!-- {{text({prompt: "このツールを理解するうえで重要なコンセプト・用語を表形式で説明してください。ソースコードから主要な概念を抽出すること。"})}} -->

| 用語 | 説明 |
| --- | --- |
| DataSource | ドキュメントテンプレートに差し込む値を返すための基底機構です。各データソースは `init()` でルートや設定を受け取り、個別の解決メソッドでマークダウン断片を返します。 |
| DocsSource | `docs/` 配下の章一覧、言語切替、前後ナビゲーションを生成するデータソースです。章ファイルのタイトル抽出や概要文の整形も担当します。 |
| ProjectSource | `package.json` からプロジェクト名、説明、バージョン、npm scripts を取得するデータソースです。scripts はマークダウン表として出力できます。 |
| PackageSource | `package.json` や `composer.json` を走査し、依存関係や scripts を解析結果として保持するスキャン対応のデータソースです。 |
| 章一覧 | `getChapterFiles` で得た章ファイル群をもとに、タイトルと概要の表を組み立てる仕組みです。概要は `## Description`、`## 説明`、`## 概要` の節から抽出されます。 |
| 言語切替 | `.sdd-forge/config.json` の言語設定と `locale/<lang>/ui.json` の表示名を使って、現在言語を強調しつつ他言語へのリンクを生成する仕組みです。 |
| 前後ナビゲーション | 現在の章ファイル名を基準に、順序付けされた章一覧から前章・次章へのリンクを生成する仕組みです。 |
| リポジトリ URL 解決 | `package.json` の `repository` から閲覧用 URL を求め、相対リンクの代わりに `blob/main/...` 形式の絶対リンクを生成する処理です。 |
| テンプレートディレクティブ | `{{data}}` や `{{text}}` により、解析済みデータや生成した本文をドキュメントへ埋め込むための記法です。 |
<!-- {{/text}} -->

### 典型的な利用フロー

<!-- {{text({prompt: "ユーザーがインストールしてから最初の成果物を得るまでの典型的な手順をステップ形式で説明してください。ソースコードのヘルプ出力やコマンド定義から手順を導出すること。"})}} -->

1. CLI ツールをインストールし、対象プロジェクトのルートに `package.json` と `docs/` を用意します。
2. 必要に応じて `.sdd-forge/config.json` を配置し、ドキュメント言語や既定言語を設定します。
3. 多言語ドキュメントを使う場合は `locale/<lang>/ui.json` を用意し、言語名の表示に必要な情報を配置します。
4. `docs/` 配下に章ファイルを置き、各章でタイトルと `## 説明` または `## 概要` の節を記述します。
5. ドキュメントテンプレート内で `{{data}}` ディレクティブを使い、章一覧、言語切替、前後ナビゲーション、プロジェクト情報を差し込める状態にします。
6. `sdd-forge build` を実行して、ソースコードと設定からナビゲーションやメタデータが反映されたドキュメントを生成します。
7. 生成結果を確認し、最初の成果物として、章一覧やリンクを含む利用者向けドキュメントを取得します。

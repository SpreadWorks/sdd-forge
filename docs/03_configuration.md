# 03. Configuration and Customization

## Overview

<!-- {{text: Write a 1–2 sentence overview of this chapter. Cover the types of configuration files, the range of configurable options, and customization points.}} -->

sdd-forge uses a set of JSON-based configuration files stored under `.sdd-forge/` to control project settings, AI provider connections, documentation output, and SDD workflow behavior. Customization spans output language, document style, preset selection, merge strategy, and AI agent definitions.
<!-- {{/text}} -->

## Contents

### Configuration Files

<!-- {{text: List all configuration files read by this tool in a table, including the location and role of each. Key files: .sdd-forge/config.json (project settings), .sdd-forge/context.json (project context), .sdd-forge/projects.json (multi-project management), .sdd-forge/current-spec (SDD flow state), .sdd-forge/output/analysis.json (analysis results), .sdd-forge/output/summary.json (lightweight version for AI).}} -->

| File | Location | Role |
|---|---|---|
| `config.json` | `.sdd-forge/config.json` | Primary project settings. Defines output languages, project type, AI providers, document style, and workflow options. Required for most commands. |
| `context.json` | `.sdd-forge/context.json` | Stores a free-text project context string that supplements AI-generated documentation. Takes precedence over `config.json`'s `textFill.projectContext`. |
| `projects.json` | `.sdd-forge/projects.json` | Registers multiple projects for management from a single sdd-forge installation. Generated by `sdd-forge setup`. |
| `current-spec` | `.sdd-forge/current-spec` | Tracks the active SDD flow state as JSON, including the spec path, base branch, and feature branch. Deleted upon flow completion. |
| `analysis.json` | `.sdd-forge/output/analysis.json` | Full source code analysis output produced by `sdd-forge scan`. Used as the authoritative data source for documentation generation. |
| `summary.json` | `.sdd-forge/output/summary.json` | Lightweight version of the analysis result optimized for AI consumption. Used in preference to `analysis.json` when present. |
<!-- {{/text}} -->

| Field | Required | Type | Default | Description |
|---|---|---|---|---|
| `output.languages` | ✅ | `string[]` | — | List of output languages for documentation (e.g., `["ja"]`, `["en", "ja"]`). |
| `output.default` | ✅ | `string` | — | The primary output language used when generating documentation. |
| `output.mode` | — | `"translate"` \| `"generate"` | `"translate"` | Method for producing non-default language output. `translate` post-processes the default language result; `generate` runs AI independently for each language. |
| `lang` | ✅ | `string` | — | Operating language for the CLI, AGENTS.md, skills, and spec templates. |
| `type` | ✅ | `string` | — | Project type determining which preset is applied (e.g., `"cli/node-cli"`, `"webapp/cakephp2"`). |
| `uiLang` | — | `"en"` \| `"ja"` | — | Language used for CLI user interface messages. |
| `documentStyle.purpose` | — | `string` | — | A short description of the documentation's intended purpose, passed to AI as context. |

| `limits.concurrency` | — | `number` | `5` | Maximum number of files processed in parallel during documentation generation. |
| `limits.designTimeoutMs` | — | `number` | — | Timeout in milliseconds for AI agent calls during the design phase. |

### Customization Points

<!-- {{text: Describe the items users can customize. (1) AI provider settings (providers field, command/args/timeoutMs/systemPromptFlag) with configuration examples, (2) document style (purpose/tone/customInstruction), (3) preset selection (type field), (4) merge strategy (flow.merge), (5) concurrency (limits.concurrency). Include JSON configuration examples for each item.}} -->

**(1) AI Provider Settings**

The `providers` field defines named AI agent configurations. Each entry specifies the executable command, argument list, timeout, and the flag used to pass a system prompt. The `{{PROMPT}}` placeholder in `args` marks where the generated prompt is injected; if omitted, the prompt is appended at the end.

```json
{
  "defaultAgent": "claude",
  "providers": {
    "claude": {
      "command": "claude",
      "args": ["--model", "claude-opus-4-5", "-p", "{{PROMPT}}"],
      "timeoutMs": 180000,
      "systemPromptFlag": "--system-prompt"
    }
  }
}
```

**(2) Document Style**

The `documentStyle` block controls the tone and intent communicated to the AI when generating `{{text}}` sections. Use `customInstruction` for any project-specific writing rules.

```json
{
  "documentStyle": {
    "purpose": "Internal technical reference for onboarding engineers",
    "tone": "formal",
    "customInstruction": "Always include a concrete code example for each concept."
  }
}
```

**(3) Preset Selection**

The `type` field selects which preset drives documentation structure and source scanning. Choose the value that best matches your project's architecture.

```json
{
  "type": "cli/node-cli"
}
```

Available values include `"cli/node-cli"`, `"webapp/cakephp2"`, `"webapp/laravel"`, `"webapp/symfony"`, and others discoverable via `sdd-forge presets`.

**(4) Merge Strategy**

The `flow.merge` field controls how feature branches are integrated into the base branch at the end of an SDD flow.

```json
{
  "flow": {
    "merge": "squash"
  }
}
```

Use `"squash"` to consolidate commits, `"ff-only"` to enforce a linear history, or `"merge"` to preserve the full branch history.

**(5) Concurrency**

The `limits.concurrency` field sets the number of files processed simultaneously during `data` and `text` generation. Increase this value on machines with sufficient resources to speed up large codebases.

```json
{
  "limits": {
    "concurrency": 10
  }
}
```

### Environment Variables

<!-- {{text: List all environment variables referenced by the tool and their purpose in a table. SDD_SOURCE_ROOT (source code root), SDD_WORK_ROOT (work root, location of .sdd-forge/), CLAUDECODE (internal variable removed to prevent Claude CLI hangs).}} -->

| Variable | Purpose |
|---|---|
| `SDD_SOURCE_ROOT` | Absolute path to the root of the source code being analyzed. When set, overrides the path resolved from `projects.json` or the current working directory. Used by scan and documentation commands to locate source files. |
| `SDD_WORK_ROOT` | Absolute path to the work root directory — the location that contains `.sdd-forge/`. Overrides the value resolved by `git rev-parse` or the current working directory. All configuration files and output artifacts are read from and written to this path. |
| `CLAUDECODE` | An internal environment variable set by the Claude CLI. sdd-forge explicitly removes this variable before spawning AI agent subprocesses via `callAgentAsync()` to prevent the Claude CLI from hanging when `stdin` is not connected to a terminal. |

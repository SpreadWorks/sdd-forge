# 03. Configuration and Customization

## Description

<!-- {{text: Write a 1–2 sentence overview of this chapter. Cover the types of configuration files, the range of configurable options, and customization points.}} -->

This chapter covers the configuration files that control `sdd-forge` behavior, from output language and AI provider settings to project type and document style. You can tailor the tool to your project's needs through a central JSON configuration file and a small set of environment variables.

## Contents

### Configuration Files

<!-- {{text: List all configuration files loaded by this tool in a table, including the location and role of each.}} -->

| File | Location | Role |
|---|---|---|
| `config.json` | `.sdd-forge/config.json` | Primary project configuration. Defines output languages, project type, AI providers, and document style. Validated on load. |
| `context.json` | `.sdd-forge/context.json` | Stores the resolved project context string used to supplement AI generation. Written by `sdd-forge` automatically; can be edited manually. |
| `projects.json` | `.sdd-forge/projects.json` | Registers one or more source projects when `sdd-forge` is run from a separate working directory. Generated by `sdd-forge setup`. |
| `current-spec` | `.sdd-forge/current-spec` | Tracks the active SDD flow state (current spec path, branch names, worktree info). Created at flow start and deleted at flow close. |
| `analysis.json` | `.sdd-forge/output/analysis.json` | Full source-code analysis result produced by `sdd-forge scan`. Used as fallback when `summary.json` is absent. |
| `summary.json` | `.sdd-forge/output/summary.json` | Lightweight version of `analysis.json`, preferred when passing data to AI agents. Generated automatically during `scan`. |

### Configuration Reference

<!-- {{text: Describe all fields in the configuration file in a table. Include field name, type, default value, and description.}} -->

All fields below belong to `.sdd-forge/config.json`. Fields marked **Required** must be present; optional fields fall back to the listed defaults.

| Field | Type | Required | Default | Description |
|---|---|---|---|---|
| `output.languages` | `string[]` | ✅ | — | List of output language codes (e.g. `["ja"]`, `["en", "ja"]`). |
| `output.default` | `string` | ✅ | — | Default output language. Must be one of the values in `output.languages`. |
| `output.mode` | `"translate"` \| `"generate"` | — | `"translate"` | How non-default language documents are produced. `translate` re-uses the default output; `generate` runs AI generation independently. |
| `lang` | `string` | ✅ | — | Language used for CLI messages, `AGENTS.md`, skills, and spec files. |
| `type` | `string` | ✅ | — | Project type identifier (e.g. `"webapp/cakephp2"`, `"cli/node-cli"`). Determines which preset and scan rules are applied. |
| `uiLang` | `"en"` \| `"ja"` | — | `"en"` | Language of the `sdd-forge` UI (prompts and status messages). |
| `documentStyle.purpose` | `string` | — | — | Describes the intended use of the generated docs (e.g. `"developer-guide"`). Passed to AI as context. |
| `documentStyle.tone` | `"polite"` \| `"formal"` \| `"casual"` | — | — | Writing tone applied to AI-generated text. |
| `documentStyle.customInstruction` | `string` | — | — | Free-form additional instruction appended to every AI text-generation prompt. |
| `textFill.projectContext` | `string` | — | — | Brief project overview used to supplement AI generation when `context.json` is absent. |
| `textFill.preamblePatterns` | `object[]` | — | — | List of regex patterns (`pattern`, `flags`) used to strip unwanted prefixes from AI output. |
| `defaultAgent` | `string` | — | — | Name of the AI provider to use when no `--agent` flag is specified. Must match a key in `providers`. |
| `providers` | `object` | — | — | Map of named AI agent definitions. Each entry specifies `command`, `args`, optional `timeoutMs`, and optional `systemPromptFlag`. |
| `flow.merge` | `"squash"` \| `"ff-only"` \| `"merge"` | — | `"squash"` | Git merge strategy used when closing an SDD flow. |
| `limits.concurrency` | `number` | — | `5` | Maximum number of files processed in parallel during `text` and `forge` commands. |
| `limits.designTimeoutMs` | `number` | — | — | Timeout in milliseconds for AI agent calls. Overrides the built-in per-command defaults. |

### Customization Points

<!-- {{text: Explain the items users can customize (providers, templates, commands, etc.). Include customization examples.}} -->

**AI Providers**

You can register any command-line AI agent under the `providers` key and select it per-run with `--agent <name>`. The `{{PROMPT}}` placeholder in `args` marks where the prompt is injected; if omitted, the prompt is appended automatically.

```json
"providers": {
  "claude": {
    "command": "claude",
    "args": ["--model", "claude-opus-4-5", "-p", "{{PROMPT}}"],
    "timeoutMs": 180000,
    "systemPromptFlag": "--system-prompt"
  }
},
"defaultAgent": "claude"
```

**Document Style**

Adjust the tone and purpose of all AI-generated documentation without touching individual templates:

```json
"documentStyle": {
  "purpose": "onboarding-guide",
  "tone": "casual",
  "customInstruction": "Always include a concrete code example for each concept."
}
```

**Project Type and Presets**

Setting `type` selects the matching preset, which determines which directories are scanned, which analyzers run, and which document templates are used. Built-in options include `webapp/cakephp2`, `webapp/laravel`, `webapp/symfony`, and `cli/node-cli`. This is the primary switch for adapting `sdd-forge` to different technology stacks.

**Output Languages**

To generate documentation in multiple languages, list all target codes in `output.languages` and choose whether additional languages are produced via AI (`generate`) or translated from the default (`translate`):

```json
"output": {
  "languages": ["ja", "en"],
  "default": "ja",
  "mode": "translate"
}
```

**Preamble Stripping**

If your AI agent consistently prepends unwanted phrases to its output, define patterns to remove them automatically:

```json
"textFill": {
  "preamblePatterns": [
    { "pattern": "^(Sure|Certainly|Of course)[,!]?\\s*", "flags": "i" }
  ]
}
```

### Environment Variables

<!-- {{text: List the environment variables referenced by the tool and their purposes in a table.}} -->

| Variable | Description |
|---|---|
| `SDD_SOURCE_ROOT` | Absolute path to the source project being analyzed. When set, overrides the path resolved from `projects.json` or the current working directory. |
| `SDD_WORK_ROOT` | Absolute path to the working repository (where `.sdd-forge/` and `docs/` reside). When set, overrides the path resolved by `git rev-parse`. |
| `CLAUDECODE` | If present in the environment, `sdd-forge` removes it before spawning AI agents to prevent Claude CLI from hanging while waiting for interactive input. |

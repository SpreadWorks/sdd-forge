# 03. Configuration and Customization

## Overview

<!-- {{text: Write a 1–2 sentence overview of this chapter. Cover the types of configuration files, the range of configurable options, and customization points.}} -->

This chapter covers all configuration files used by sdd-forge, the full range of configurable options available in `.sdd-forge/config.json`, and the customization points for AI providers, document style, project types, and workflow behavior. Understanding these settings allows you to tailor documentation generation, AI agent integration, and the SDD workflow to fit your project's specific needs.
<!-- {{/text}} -->

## Contents

### Configuration Files

<!-- {{text: List all configuration files read by this tool in a table, including the location and role of each. Key files: .sdd-forge/config.json (project settings), .sdd-forge/context.json (project context), .sdd-forge/projects.json (multi-project management), .sdd-forge/current-spec (SDD flow state), .sdd-forge/output/analysis.json (analysis results), .sdd-forge/output/summary.json (lightweight version for AI).}} -->

| File | Location | Role |
|---|---|---|
| `config.json` | `.sdd-forge/config.json` | Primary project configuration. Defines output languages, project type, AI agent settings, document style, and workflow behavior. Required for most commands. |
| `context.json` | `.sdd-forge/context.json` | Stores a free-text project context string used as supplementary information for AI-generated documentation. Created and updated via `saveContext()`. |
| `projects.json` | `.sdd-forge/projects.json` | Registers multiple projects for use with sdd-forge. Generated by the `setup` command. Includes source path, work root, and a default project designation. |
| `current-spec` | `.sdd-forge/current-spec` | Tracks the active SDD flow state as JSON. Records the current spec path, base branch, feature branch, and worktree details. Deleted on flow completion. |
| `analysis.json` | `.sdd-forge/output/analysis.json` | Full source code analysis result produced by `sdd-forge scan`. Stored without indentation. Used as the authoritative data source for documentation generation. |
| `summary.json` | `.sdd-forge/output/summary.json` | Lightweight version of `analysis.json` optimized for AI consumption. Commands that invoke AI agents prefer this file; they fall back to `analysis.json` if it is absent. |
<!-- {{/text}} -->

### Configuration Reference

| Field | Required | Type | Default | Description |
|---|---|---|---|---|
| `output.languages` | ✅ | `string[]` | — | List of output languages for generated documentation (e.g., `["ja"]`, `["en", "ja"]`). |
| `output.default` | ✅ | `string` | — | The primary output language. Documentation is generated in this language first. |
| `output.mode` | — | `"translate"` \| `"generate"` | `"translate"` | How non-default languages are produced. `translate` runs the translation pipeline; `generate` calls AI independently for each language. |
| `lang` | ✅ | `string` | — | Operating language for the CLI, AGENTS.md, skills, and spec files. |
| `type` | ✅ | `string` | — | Project type identifier, which determines which preset is applied (e.g., `"cli/node-cli"`, `"webapp/cakephp2"`). |
| `uiLang` | — | `"en"` \| `"ja"` | — | Display language for UI messages shown to the operator. |
| `documentStyle.purpose` | — | `string` | — | Describes the intended audience and purpose of the generated documentation, used to guide AI output. |
| `documentStyle.tone` | — | `"polite"` \| `"formal"` \| `"casual"` | — | Sets the writing tone applied to AI-generated text sections. |
| `documentStyle.customInstruction` | — | `string` | — | Additional free-text instructions passed to the AI during text generation. |
| `textFill.projectContext` | — | `string` | — | A project overview string injected as context for AI generation. Overridden by `context.json` if present. |
| `textFill.preamblePatterns` | — | `object[]` | — | List of patterns used to strip unwanted preamble prefixes from LLM output. |

| `limits.concurrency` | — | `number` | `5` | Number of files processed in parallel during documentation generation. |
| `limits.designTimeoutMs` | — | `number` | — | Maximum time in milliseconds allowed for a single AI agent call during documentation generation. |

### Customization Points

<!-- {{text: Describe the items users can customize. (1) AI provider settings (providers field, command/args/timeoutMs/systemPromptFlag) with configuration examples, (2) document style (purpose/tone/customInstruction), (3) preset selection (type field), (4) merge strategy (flow.merge), (5) concurrency (limits.concurrency). Include JSON configuration examples for each item.}} -->

**1. AI Provider Settings**

The `providers` field defines named AI agent configurations. Each provider specifies the command to invoke, its arguments, an optional timeout, and how to pass the system prompt. The `defaultAgent` field selects which provider is used by default.

```json
{
  "defaultAgent": "claude",
  "providers": {
    "claude": {
      "command": "claude",
      "args": ["--model", "claude-opus-4-5", "{{PROMPT}}"],
      "timeoutMs": 120000,
      "systemPromptFlag": "--system-prompt"
    }
  }
}
```

The `{{PROMPT}}` placeholder in `args` marks where the generated prompt is injected. If omitted, the prompt is appended to the end of the argument list. Set `systemPromptFlag` to `"--system-prompt-file"` when the agent requires the system prompt to be written to a temporary file.

**2. Document Style**

The `documentStyle` block controls how AI-generated text sections are written. You can specify the purpose and intended audience, the overall tone, and any additional free-text instructions.

```json
{
  "documentStyle": {
    "purpose": "Internal reference for onboarding engineers",
    "tone": "formal",
    "customInstruction": "Use concrete code examples wherever possible."
  }
}
```

**3. Preset Selection**

The `type` field selects the preset applied to the project, which determines which source files are scanned and which documentation templates are used.

```json
{
  "type": "cli/node-cli"
}
```

Available values include `"webapp/cakephp2"`, `"webapp/laravel"`, `"webapp/symfony"`, and `"cli/node-cli"`. Type aliases defined in `lib/types.js` are also accepted (e.g., `"php-mvc"` as an alias for `"webapp/cakephp2"`).

**4. Merge Strategy**

The `flow.merge` field controls how feature branches are merged into the base branch at the end of the SDD workflow.

```json
{
  "flow": {
    "merge": "squash"
  }
}
```

Use `"squash"` to combine all feature commits into a single commit, `"ff-only"` to require a fast-forward merge, or `"merge"` to create an explicit merge commit.

**5. Concurrency**

The `limits.concurrency` field sets the number of files that are processed in parallel during documentation generation. Lowering this value reduces memory and API usage; raising it speeds up generation on large codebases.

```json
{
  "limits": {
    "concurrency": 3
  }
}
```

### Environment Variables

<!-- {{text: List all environment variables referenced by the tool and their purpose in a table. SDD_SOURCE_ROOT (source code root), SDD_WORK_ROOT (work root, location of .sdd-forge/), CLAUDECODE (internal variable removed to prevent Claude CLI hangs).}} -->

| Variable | Purpose |
|---|---|
| `SDD_SOURCE_ROOT` | Absolute path to the root of the source code being analyzed. When set, `sourceRoot()` uses this value directly instead of falling back to the repository root. Set automatically by the project context resolution logic in `sdd-forge.js`. |
| `SDD_WORK_ROOT` | Absolute path to the work root directory, which is the directory containing `.sdd-forge/`. When set, `repoRoot()` uses this value instead of running `git rev-parse`. Set automatically during project context resolution. |
| `CLAUDECODE` | An internal environment variable that is actively removed before spawning AI agent subprocesses. Its presence can cause the Claude CLI to hang waiting for input; deleting it from the child process environment prevents this issue. |

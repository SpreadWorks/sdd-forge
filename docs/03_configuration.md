# 03. Configuration and Customization

## Overview

<!-- {{text: Write a 1–2 sentence overview of this chapter. Cover the types of configuration files, the range of configurable options, and customization points.}} -->

This chapter covers all configuration files used by sdd-forge, the full range of available settings in `.sdd-forge/config.json`, and how to customize AI providers, document style, and workflow behavior to suit your project's needs.
<!-- {{/text}} -->

## Contents

### Configuration Files

<!-- {{text: List all configuration files read by this tool in a table, including the location and role of each. Key files: .sdd-forge/config.json (project settings), .sdd-forge/context.json (project context), .sdd-forge/projects.json (multi-project management), .sdd-forge/current-spec (SDD flow state), .sdd-forge/output/analysis.json (analysis results), .sdd-forge/output/summary.json (lightweight version for AI).}} -->

| File | Role |
|---|---|
| `.sdd-forge/config.json` | Primary project configuration file. Defines output languages, project type, AI providers, document style, and workflow settings. |
| `.sdd-forge/context.json` | Stores the project context string passed to AI during document generation. Takes precedence over `textFill.projectContext` in `config.json`. |
| `.sdd-forge/projects.json` | Registers multiple projects for management from a single sdd-forge installation. Generated by the `setup` command. |
| `.sdd-forge/current-spec` | Tracks the active SDD flow state as a JSON object, including the current spec path, branch names, and worktree information. Deleted upon flow completion. |
| `.sdd-forge/output/analysis.json` | Full source code analysis results generated by `sdd-forge scan`. Stored without indentation. |
| `.sdd-forge/output/summary.json` | Lightweight version of `analysis.json` optimized for AI consumption. Used in preference to `analysis.json` wherever available. |
<!-- {{/text}} -->

### Configuration Reference

| Field | Required | Type | Default | Description |
|---|---|---|---|---|
| `output.languages` | ✅ | `string[]` | — | List of output languages for generated documentation (e.g., `["ja"]`, `["en", "ja"]`). |
| `output.default` | ✅ | `string` | — | The primary output language. Documentation is generated in this language first. |
| `output.mode` | — | `"translate"` \| `"generate"` | `"translate"` | How non-default languages are produced: `translate` uses the default-language output as the source, `generate` produces each language independently. |
| `lang` | ✅ | `string` | — | Operating language for the CLI, AGENTS.md, skills, and spec files. |
| `type` | ✅ | `string` | — | Project type that determines which preset is applied (e.g., `"webapp/cakephp2"`, `"cli/node-cli"`). |
| `uiLang` | — | `"en"` \| `"ja"` | — | Language used for CLI user interface messages. |
| `documentStyle.purpose` | — | `string` | — | Describes the intended purpose of the generated documentation, used as guidance for AI text generation. |
| `documentStyle.tone` | — | `"polite"` \| `"formal"` \| `"casual"` | — | Writing tone applied to AI-generated documentation text. |

| `flow.merge` | — | `"squash"` \| `"ff-only"` \| `"merge"` | `"squash"` | Merge strategy applied when completing an SDD flow and merging the feature branch. |
| `limits.concurrency` | — | `number` | `5` | Maximum number of files processed in parallel during documentation generation. |
| `limits.designTimeoutMs` | — | `number` | — | Timeout in milliseconds for AI agent calls during document generation. |

### Customization Points

<!-- {{text: Describe the items users can customize. (1) AI provider settings (providers field, command/args/timeoutMs/systemPromptFlag) with configuration examples, (2) document style (purpose/tone/customInstruction), (3) preset selection (type field), (4) merge strategy (flow.merge), (5) concurrency (limits.concurrency). Include JSON configuration examples for each item.}} -->

#### 1. AI Provider Settings

The `providers` field defines named AI agent configurations. Each entry specifies how sdd-forge invokes the external AI command. Use `{{PROMPT}}` as a placeholder in `args` to control where the prompt is injected; if omitted, the prompt is appended at the end.

```json
{
  "defaultAgent": "claude",
  "providers": {
    "claude": {
      "command": "claude",
      "args": ["--output-format", "text", "-p", "{{PROMPT}}"],
      "timeoutMs": 120000,
      "systemPromptFlag": "--system-prompt"
    }
  }
}
```

#### 2. Document Style

Control the tone and purpose of AI-generated documentation text. The `customInstruction` field accepts any free-form guidance.

```json
{
  "documentStyle": {
    "purpose": "Internal developer reference for onboarding and maintenance",
    "tone": "formal",
    "customInstruction": "Avoid marketing language. Focus on technical accuracy."
  }
}
```

#### 3. Preset Selection

The `type` field selects the preset that determines scan targets, documentation structure, and framework-specific analysis. Use the canonical preset key for your project type.

```json
{
  "type": "cli/node-cli"
}
```

Available values include `"webapp/cakephp2"`, `"webapp/laravel"`, `"webapp/symfony"`, and `"cli/node-cli"`.

#### 4. Merge Strategy

The `flow.merge` field controls how feature branches are merged into the base branch at the end of an SDD flow.

```json
{
  "flow": {
    "merge": "squash"
  }
}
```

| Value | Behavior |
|---|---|
| `squash` | Squashes all commits into one before merging (default) |
| `ff-only` | Fast-forward merge only; fails if not possible |
| `merge` | Standard merge commit |

#### 5. Concurrency

Tune parallel file processing to match your environment's capabilities or rate limits.

```json
{
  "limits": {
    "concurrency": 3,
    "designTimeoutMs": 180000
  }
}
```

### Environment Variables

<!-- {{text: List all environment variables referenced by the tool and their purpose in a table. SDD_SOURCE_ROOT (source code root), SDD_WORK_ROOT (work root, location of .sdd-forge/), CLAUDECODE (internal variable removed to prevent Claude CLI hangs).}} -->

| Variable | Purpose |
|---|---|
| `SDD_SOURCE_ROOT` | Absolute path to the source code root of the target project. Used by scan and analysis commands to locate source files. When set, takes precedence over the path derived from git or the current working directory. |
| `SDD_WORK_ROOT` | Absolute path to the work root — the directory that contains the `.sdd-forge/` folder. When set, overrides the git repository root and current working directory as the location for configuration and output files. |
| `CLAUDECODE` | An internal environment variable that is explicitly removed before spawning AI agent subprocesses. This prevents Claude CLI from hanging due to inherited session state. |

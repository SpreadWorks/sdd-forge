# Configuration and Customization

## Description

<!-- {{text({prompt: "Write a 1-2 sentence overview of this chapter. Include the types of config files, range of configurable items, and customization points."})}} -->

sdd-forge uses a project-level configuration file (`.sdd-forge/config.json`) and environment variables to control documentation generation behavior. Configurable items range from output languages and document style to scan targets, AI agent providers, parallel processing concurrency, and SDD flow merge strategy.

<!-- {{/text}} -->

## Content

### Configuration Files

<!-- {{text({prompt: "List all configuration files this tool reads, including their locations and roles, in table format. Extract from file reading logic in the source code."})}} -->

| File | Location | Role |
|---|---|---|
| `config.json` | `.sdd-forge/config.json` | Main project configuration. Defines language, preset type, documentation settings, agent providers, scan patterns, and flow options. Created by `sdd-forge setup`. |
| `package.json` | `<project root>/package.json` | Read via `loadPackageField()` to extract project metadata such as name and version. |
| `preset.json` | `src/presets/<key>/preset.json` | Defines each preset's parent chain, label, scan patterns, and chapter order. Discovered automatically at startup by `discoverPresets()`. |
| `analysis.json` | `.sdd-forge/output/analysis.json` | Generated by `sdd-forge scan`. Stores the source code analysis results consumed by downstream pipeline steps (enrich, data, text). |

### Customization Points

<!-- {{text({prompt: "Describe items that users can customize. Extract configurable items from the source code and include configuration examples for each.", mode: "deep"})}} -->

**Document Style**

Control the purpose and tone of generated documentation. The `docs.style` object is passed to the AI text generation step.

```json
{
  "docs": {
    "style": {
      "purpose": "user-guide",
      "tone": "polite",
      "customInstruction": "Use short sentences. Avoid jargon."
    }
  }
}
```

**Multi-language Output**

Generate documentation in multiple languages. The `defaultLanguage` is generated first; additional languages are produced according to the `mode` setting.

```json
{
  "docs": {
    "languages": ["en", "ja"],
    "defaultLanguage": "en",
    "mode": "translate"
  }
}
```

**Preset Composition**

Combine multiple presets by specifying an array in `type`. Each preset's scan patterns and DataSources are merged through their parent chains. Parent-child duplicates are automatically deduplicated.

```json
{
  "type": ["symfony", "postgres"]
}
```

**Chapter Order**

Override the default chapter sequence defined by the preset. Chapters listed here control the order of generated documentation files.

```json
{
  "chapters": ["overview.md", "configuration.md", "cli_commands.md", "internal_design.md"]
}
```

**Scan Patterns**

Customize which files are included or excluded from source code analysis. These patterns override the preset's default scan configuration.

```json
{
  "scan": {
    "include": ["src/**/*.ts", "lib/**/*.js"],
    "exclude": ["**/*.test.ts", "dist/**"]
  }
}
```

**Parallel Processing**

Adjust the number of concurrent file processing tasks. Higher values speed up scanning on large codebases but consume more memory.

```json
{
  "concurrency": 10
}
```

**AI Agent Provider**

Configure which AI agent to use, its timeout, and working directory. The `providers` object defines the command, arguments, and optional system prompt flag for each provider.

```json
{
  "agent": {
    "default": "claude",
    "workDir": ".tmp",
    "timeout": 600,
    "providers": {
      "claude": {
        "name": "Claude",
        "command": "claude",
        "args": ["-p", "{{PROMPT}}"]
      }
    }
  }
}
```

**Flow Merge Strategy**

Choose how SDD feature branches are merged back into the base branch.

```json
{
  "flow": {
    "merge": "squash",
    "push": { "remote": "origin" }
  }
}
```

**GitHub CLI Integration**

Enable GitHub CLI (`gh`) integration for SDD flow operations such as pull request creation.

```json
{
  "commands": {
    "gh": "enable"
  }
}
```

### Environment Variables

<!-- {{text({prompt: "List all environment variables referenced by the tool and their purposes in table format. Extract from process.env references in the source code.", mode: "deep"})}} -->

| Variable | Purpose | Set By |
|---|---|---|
| `SDD_WORK_ROOT` | Specifies the work directory (the parent of `.sdd-forge/` and `docs/`). When set, `repoRoot()` returns this value directly instead of running `git rev-parse --show-toplevel`. | `sdd-forge.js` at startup, or set manually by the user. |
| `SDD_SOURCE_ROOT` | Specifies the source code root to scan. When set, `sourceRoot()` returns this value. When unset, it falls back to the same value as `SDD_WORK_ROOT`. Useful when the source tree and work tree are in different locations. | `sdd-forge.js` at startup, or set manually by the user. |
| `CLAUDECODE` | Detected and removed from the environment before invoking AI agents (via `agent.js`). This prevents internal Claude Code state from leaking into child processes. | Set externally by the Claude Code runtime; not intended for user configuration. |

If neither `SDD_WORK_ROOT` nor `SDD_SOURCE_ROOT` is set, the tool resolves the project root by running `git rev-parse --show-toplevel`. If that also fails (e.g. outside a Git repository), it falls back to `process.cwd()`.

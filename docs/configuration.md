# Configuration and Customization

## Description

<!-- {{text({prompt: "Write a 1-2 sentence overview of this chapter. Include the types of config files, range of configurable items, and customization points."})}} -->

sdd-forge reads its project configuration from `.sdd-forge/config.json` and `package.json`, covering documentation output, language settings, preset types, AI agent providers, scan targets, flow strategies, and external command availability. Users can customize everything from document style and chapter order to concurrency limits and merge strategies, with additional runtime control through environment variables.
<!-- {{/text}} -->

## Content

### Configuration Files

<!-- {{text({prompt: "List all configuration files this tool reads, including their locations and roles, in table format. Extract from file reading logic in the source code."})}} -->

| File | Location | Role |
|------|----------|------|
| `config.json` | `.sdd-forge/config.json` | Primary project configuration. Defines documentation settings, language, preset type, agent providers, flow options, and more. Created by `sdd-forge setup`. |
| `package.json` | Project root | Provides project metadata. sdd-forge reads arbitrary fields (e.g., `name`) via `loadPackageField()` for documentation generation. |
| `analysis.json` | `.sdd-forge/output/analysis.json` | Generated by `sdd-forge scan`. Stores the parsed source-code analysis used by downstream commands. |
| `preset.json` | Bundled in `src/presets/<type>/` | Defines preset-specific defaults such as chapter order, scan patterns, and data sources. Resolved via the `type` field in `config.json`. |
| `wrangler.toml` / `wrangler.json` | Project root (Workers preset) | Cloudflare Workers binding configuration. Parsed for KV, R2, D1, service bindings, env vars, and runtime constraints. |
| `composer.json` | Project root (PHP presets) | Read by Symfony, Laravel, and CakePHP presets to extract dependency lists. |
| `.env` / `.env.local` | Project root (webapp presets) | Parsed by webapp presets for environment variable definitions. |

### Customization Points

<!-- {{text({prompt: "Describe items that users can customize. Extract configurable items from the source code and include configuration examples for each.", mode: "deep"})}} -->

**Document Style**

Control the tone and purpose of generated documentation through `docs.style`. This affects how AI generates text content for your docs.

```json
{
  "docs": {
    "languages": ["en"],
    "defaultLanguage": "en",
    "style": {
      "purpose": "user-guide",
      "tone": "polite",
      "customInstruction": "Focus on practical examples and avoid jargon."
    }
  }
}
```

**Multi-Language Output**

Generate documentation in multiple languages. The `mode` field controls whether non-default languages are produced by translation or independent generation.

```json
{
  "docs": {
    "languages": ["en", "ja"],
    "defaultLanguage": "en",
    "mode": "translate"
  }
}
```

**Chapter Order**

Override the preset's default chapter sequence by specifying a `chapters` array in your config. Each entry corresponds to a chapter file name (without the number prefix).

```json
{
  "chapters": ["overview", "architecture", "cli_commands", "configuration"]
}
```

**Scan Targets**

Customize which files are included or excluded from source analysis by defining `scan.include` and `scan.exclude` glob patterns.

```json
{
  "scan": {
    "include": ["src/**/*.ts", "lib/**/*.js"],
    "exclude": ["**/*.test.ts", "**/fixtures/**"]
  }
}
```

**Concurrency**

Adjust the number of files processed in parallel. Increase for faster builds on powerful machines, or decrease to reduce resource usage.

```json
{
  "concurrency": 10
}
```

**Agent Providers**

Configure AI agent providers for text generation and enrichment. The `{{PROMPT}}` placeholder in `args` is replaced with the actual prompt at runtime.

```json
{
  "agent": {
    "default": "claude",
    "timeout": 120,
    "retryCount": 3,
    "providers": {
      "claude": {
        "name": "Claude",
        "command": "claude",
        "args": ["-p", "{{PROMPT}}"]
      }
    }
  }
}
```

**Flow Merge Strategy**

Choose how SDD flow branches are merged back into the main branch.

```json
{
  "flow": {
    "merge": "ff-only",
    "push": {
      "remote": "origin"
    }
  }
}
```

**GitHub CLI Integration**

Enable GitHub CLI (`gh`) integration to allow sdd-forge to create pull requests during the flow finalize step.

```json
{
  "commands": {
    "gh": "enable"
  }
}
```

### Environment Variables

<!-- {{text({prompt: "List all environment variables referenced by the tool and their purposes in table format. Extract from process.env references in the source code.", mode: "deep"})}} -->

| Variable | Purpose |
|----------|--------|
| `SDD_WORK_ROOT` | Specifies the working directory where `.sdd-forge/` and `docs/` reside. When set, sdd-forge uses this path instead of auto-detecting the git repository root. Used in project mode when the working directory differs from the source root. |
| `SDD_SOURCE_ROOT` | Specifies the source code root directory to analyze. When set, sdd-forge reads source files from this path rather than the working root. This allows the source tree and the documentation output to live in separate locations. |

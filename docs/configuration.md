# Configuration and Customization

## Description

<!-- {{text({prompt: "Write a 1-2 sentence overview of this chapter. Include the types of config files, range of configurable items, and customization points."})}} -->

sdd-forge uses two primary configuration files ‚Äî `.sdd-forge/config.json` for project-specific settings and `preset.json` for framework-specific scan and template definitions. Together, these files control documentation output languages, AI agent behavior, scan targets, chapter ordering, flow strategies, and preset inheritance, providing a wide range of customization points to tailor the tool to any project.
<!-- {{/text}} -->

## Content

### Configuration Files

<!-- {{text({prompt: "List all configuration files this tool reads, including their locations and roles, in table format. Extract from file reading logic in the source code."})}} -->

| File | Location | Role |
| --- | --- | --- |
| `config.json` | `.sdd-forge/config.json` | Main project configuration. Stores documentation settings, language, preset type, agent configuration, scan overrides, flow strategy, and concurrency. Created by `sdd-forge setup`. |
| `preset.json` | `src/presets/<key>/preset.json` | Preset definition for each framework. Defines parent chain, label, aliases, scan include/exclude patterns, and chapter ordering. Discovered automatically at startup. |
| `package.json` | Project root | Read for project metadata (e.g., name, version). Individual fields are loaded on demand via `loadPackageField()`. |
| `analysis.json` | `.sdd-forge/output/analysis.json` | Generated by `sdd-forge scan`. Contains the parsed source code analysis used by downstream pipeline steps (enrich, data, text). |
| `flow.json` | `.sdd-forge/flow.json` | Persists SDD flow state (current step, branch, request) for the active development flow. |

### Customization Points

<!-- {{text({prompt: "Describe items that users can customize. Extract configurable items from the source code and include configuration examples for each.", mode: "deep"})}} -->

**Documentation Language and Style**

Control which languages your documentation is generated in, and set the writing tone. The `docs.style` object lets you adjust the purpose and tone of generated text.

```json
{
  "docs": {
    "languages": ["en", "ja"],
    "defaultLanguage": "en",
    "mode": "translate",
    "style": {
      "purpose": "developer-guide",
      "tone": "polite",
      "customInstruction": "Use short paragraphs and bullet lists."
    }
  }
}
```

**Preset Type Selection**

Specify one or more framework presets to activate the appropriate scan patterns, DataSources, and chapter templates. Multiple presets can be combined when a project uses several technologies.

```json
{
  "type": ["laravel", "drizzle"]
}
```

**Scan Target Customization**

Override the preset's default scan include/exclude patterns at the project level to focus on specific directories or skip irrelevant files.

```json
{
  "scan": {
    "include": ["src/**/*.ts", "lib/**/*.ts"],
    "exclude": ["src/generated/**"]
  }
}
```

**Chapter Ordering**

Override the preset's default chapter order to rearrange the documentation structure for your project's needs.

```json
{
  "chapters": ["overview.md", "configuration.md", "cli_commands.md", "architecture.md"]
}
```

**AI Agent Configuration**

Define which AI agent to use, its timeout, and custom provider commands. Each provider specifies a command, arguments (with `{{PROMPT}}` placeholder), and optional timeout.

```json
{
  "agent": {
    "default": "claude",
    "timeout": 300,
    "providers": {
      "claude": {
        "name": "Claude",
        "command": "claude",
        "args": ["-p", "{{PROMPT}}"]
      }
    }
  }
}
```

**Concurrency**

Adjust the number of parallel file-processing tasks. The default is 5.

```json
{
  "concurrency": 10
}
```

**Flow and Merge Strategy**

Configure how SDD flow branches are merged and whether automatic push is performed after finalization.

```json
{
  "flow": {
    "merge": "squash",
    "push": { "remote": "origin" }
  },
  "commands": { "gh": "enable" }
}
```

### Environment Variables

<!-- {{text({prompt: "List all environment variables referenced by the tool and their purposes in table format. Extract from process.env references in the source code.", mode: "deep"})}} -->

| Variable | Purpose |
| --- | --- |
| `SDD_WORK_ROOT` | Overrides the working directory (the parent of `.sdd-forge/` and `docs/`). When set, `repoRoot()` returns this value instead of detecting the Git repository root. Used in project mode and test environments. |
| `SDD_SOURCE_ROOT` | Overrides the source code root directory. When set, `sourceRoot()` returns this value instead of falling back to `repoRoot()`. Allows the source tree and working directory to be separated (e.g., monorepos or cross-project documentation). |

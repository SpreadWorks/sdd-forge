# 03. Configuration and Customization

## Description

<!-- {{text: Write a 1-2 sentence overview of this chapter. Include the types of config files, range of configurable items, and customization points.}} -->

sdd-forge is configured primarily through `.sdd-forge/config.json`, which controls output language, project type, document style, AI agent providers, scan scope, and SDD flow behavior. Preset-specific settings are defined in `preset.json` files, and two environment variables allow overriding project root detection for advanced use cases.

<!-- {{/text}} -->

## Content

### Configuration Files

<!-- {{text: List all configuration files this tool reads, including their locations and roles, in table format. Extract from file reading logic in the source code.}} -->

| File | Location | Role |
|------|----------|------|
| `config.json` | `.sdd-forge/config.json` | Primary project configuration. Defines output language, project type, document style, agent providers, scan scope, concurrency limits, and flow settings. Created by `sdd-forge setup`. |
| `preset.json` | `src/presets/<key>/preset.json` | Preset manifest. Declares the architecture layer, label, aliases, scan patterns, and chapter ordering for each preset (e.g., `cakephp2`, `laravel`, `symfony`). Discovered automatically at startup. |
| `package.json` | Project root | Read via `loadPackageField()` to extract project metadata such as the package name or version. |
| `analysis.json` | `.sdd-forge/output/analysis.json` | Generated by `sdd-forge scan`. Stores the source code analysis results consumed by downstream commands (`enrich`, `data`, `text`). |

The configuration directory `.sdd-forge/` also contains two subdirectories: `output/` for build artifacts and `data/` for supplementary data files used during document generation.

### Configuration Reference

<!-- {{text[mode=deep]: Describe all configuration fields in table format. Include field name, required/optional, type, default value, and description. Extract from validation logic and default value definitions in the source code.}} -->

The table below lists every field recognized in `.sdd-forge/config.json`. Fields marked **required** cause a validation error if omitted.

| Field | Required | Type | Default | Description |
|-------|----------|------|---------|-------------|
| `output` | Yes | Object | — | Output language configuration. |
| `output.languages` | Yes | `string[]` | — | Language codes for document output (e.g., `["en"]`, `["en", "ja"]`). |
| `output.default` | Yes | `string` | — | Default output language. Must be included in `output.languages`. |
| `output.mode` | No | `"translate"` \| `"generate"` | `"translate"` | How non-default languages are produced. |
| `lang` | Yes | `string` | — | Operating language for CLI messages, AGENTS.md, skills, and specs. Falls back to `"en"` when read before validation. |
| `type` | Yes | `string` | — | Project type identifier. Short aliases (e.g., `"cakephp2"`) are resolved to full paths (e.g., `"webapp/cakephp2"`) automatically. |
| `limits` | No | Object | — | Resource limit settings. |
| `limits.agentTimeout` | No | `number` | `300` | Agent call timeout in seconds. |
| `limits.concurrency` | No | `number` | `5` | Maximum parallel file processing tasks. |
| `documentStyle` | No | Object | — | Controls the style of generated documentation. |
| `documentStyle.purpose` | Yes (if object provided) | `string` | — | Document purpose: `"developer-guide"`, `"user-guide"`, `"api-reference"`, or a custom string. |
| `documentStyle.tone` | Yes (if object provided) | `"polite"` \| `"formal"` \| `"casual"` | — | Writing tone for generated text. |
| `documentStyle.customInstruction` | No | `string` | — | Additional free-form instruction appended to the document style prompt. |
| `textFill` | No | Object | — | Settings for AI text generation. |
| `textFill.projectContext` | No | `string` | — | A project overview paragraph provided to the AI during text generation. |
| `textFill.preamblePatterns` | No | `PreamblePattern[]` | `[{ "pattern": "^(Here is\|以下に\|Based on)", "flags": "i" }]` | Regex patterns stripped from the beginning of AI-generated output. |
| `agentWorkDir` | No | `string` | `".tmp"` | Working directory used when spawning agent subprocesses. |
| `chapters` | No | `string[]` | (from preset) | Custom chapter ordering. Each entry is a filename without its extension (e.g., `["overview", "architecture"]`). Overrides the preset default. |
| `scan` | No | Object | — | Source code scanning configuration. |
| `scan.include` | Yes (if object provided) | `string[]` | — | Glob patterns for files to include in the scan. |
| `scan.exclude` | No | `string[]` | — | Glob patterns for files to exclude from the scan. |
| `flow` | No | Object | — | SDD flow behavior. |
| `flow.merge` | No | `"squash"` \| `"ff-only"` \| `"merge"` | `"squash"` | Git merge strategy used when closing a flow. |
| `defaultAgent` | No | `string` | — | Default agent provider key (e.g., `"claude"`). |
| `providers` | No | `Object` | — | Named agent provider definitions (see below). |
| `providers.<name>.command` | Yes (if provider defined) | `string` | — | Executable command for the agent (e.g., `"claude"`, `"codex"`). |
| `providers.<name>.args` | Yes (if provider defined) | `string[]` | — | Command-line arguments. Use the `{{PROMPT}}` placeholder for prompt injection. |
| `providers.<name>.name` | No | `string` | — | Display name for the provider. |
| `providers.<name>.systemPromptFlag` | No | `string` | — | CLI flag for passing a system prompt (e.g., `"--system-prompt"`). When omitted, the system prompt is prepended to the user prompt. |
| `providers.<name>.timeoutMs` | No | `number` | — | Per-invocation timeout in milliseconds. |

Validation is performed by `validateConfig()` in `src/lib/types.js`. When multiple errors are detected, they are collected and reported together in a single exception.

### Customization Points

<!-- {{text[mode=deep]: Describe items that users can customize. Extract configurable items from the source code and include configuration examples for each.}} -->

**Document Style**

Control the purpose, tone, and additional instructions for AI-generated documentation. The `purpose` value appears in the system prompt sent to the agent, so it directly influences the writing style.

```json
{
  "documentStyle": {
    "purpose": "user-guide",
    "tone": "casual",
    "customInstruction": "Use short sentences and include practical examples."
  }
}
```

**Chapter Ordering**

Override the default chapter sequence defined by the preset. Each entry corresponds to a Markdown filename in the `docs/` directory (without extension).

```json
{
  "chapters": ["overview", "architecture", "configuration", "cli_commands", "data_model"]
}
```

**Scan Scope**

Restrict which source files are analyzed. The `include` patterns determine the scan targets, while `exclude` patterns filter out unwanted files such as tests or vendor code.

```json
{
  "scan": {
    "include": ["src/**/*.js", "lib/**/*.ts"],
    "exclude": ["**/*.test.js", "**/node_modules/**"]
  }
}
```

**Agent Providers**

Register one or more AI agent backends. The `{{PROMPT}}` placeholder in `args` is replaced with the actual prompt at invocation time. If the assembled command-line exceeds 100 KB, the prompt is automatically routed through stdin instead.

```json
{
  "defaultAgent": "claude",
  "providers": {
    "claude": {
      "name": "claude-cli",
      "command": "claude",
      "args": ["--model", "sonnet", "-p", "{{PROMPT}}"],
      "systemPromptFlag": "--system-prompt"
    },
    "codex": {
      "name": "codex-cli",
      "command": "codex",
      "args": ["exec", "--full-auto", "-C", ".tmp", "{{PROMPT}}"]
    }
  }
}
```

**Concurrency and Timeouts**

Tune parallel processing and agent call durations to match your environment. Lower concurrency values reduce memory usage on constrained machines.

```json
{
  "limits": {
    "concurrency": 3,
    "agentTimeout": 600
  }
}
```

**Preamble Stripping**

AI models sometimes prepend conversational phrases to their output. Define regex patterns to automatically remove these prefixes from generated text.

```json
{
  "textFill": {
    "preamblePatterns": [
      { "pattern": "^(Here is|以下に|Based on|Sure,)", "flags": "i" }
    ]
  }
}
```

**Flow Merge Strategy**

Choose how feature branches are merged back when closing an SDD flow. The default `"squash"` combines all commits into one.

```json
{
  "flow": {
    "merge": "ff-only"
  }
}
```

### Environment Variables

<!-- {{text[mode=deep]: List all environment variables referenced by the tool and their purposes in table format. Extract from process.env references in the source code.}} -->

sdd-forge reads two environment variables to override automatic project root detection. These are primarily used in CI pipelines or when the `.sdd-forge/` directory and the source code reside in separate locations.

| Variable | Purpose | Fallback Behavior |
|----------|---------|-------------------|
| `SDD_WORK_ROOT` | Overrides the work root where `.sdd-forge/` is located. When set, the tool skips git repository detection and uses this path directly. | `git rev-parse --show-toplevel`, then `process.cwd()` |
| `SDD_SOURCE_ROOT` | Overrides the source code root used for scanning. Allows the analyzed source tree to differ from the work root. | Value of `SDD_WORK_ROOT` (or its fallback chain) |

The resolution order for the work root is: `SDD_WORK_ROOT` → git repository root → current working directory. The source root follows: `SDD_SOURCE_ROOT` → work root.

One additional variable is handled internally: `CLAUDECODE` is explicitly removed from the environment before spawning agent subprocesses, preventing nested Claude CLI sessions from inheriting the parent session's authentication context.

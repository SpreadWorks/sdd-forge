# 03. Configuration and Customization

## Description

<!-- {{text: Write a 1-2 sentence overview of this chapter. Include the types of config files, range of configurable items, and customization points.}} -->

sdd-forge is configured primarily through `.sdd-forge/config.json` (project-level settings) and `preset.json` (framework-level defaults). These files control documentation output languages, project type detection, AI agent invocation, document styling, scan targets, chapter ordering, concurrency, and SDD flow behavior.

<!-- {{/text}} -->

## Content

### Configuration Files

<!-- {{text: List all configuration files this tool reads, including their locations and roles, in table format. Extract from file reading logic in the source code.}} -->

| File | Location | Role |
|---|---|---|
| `config.json` | `.sdd-forge/config.json` | Primary project configuration. Defines language, project type, documentation settings, AI agent providers, flow options, and scan patterns. Created by `sdd-forge setup`. |
| `preset.json` | `src/presets/<preset>/preset.json` | Framework preset definition. Declares parent chain, language layer, chapter ordering, scan include/exclude patterns, aliases, and labels. Bundled with the package. |
| `flow.json` | `.sdd-forge/flow.json` | SDD workflow state persistence. Tracks the current flow's spec path, branch names, step progress, requirements, and notes. Managed automatically by flow commands. |
| `package.json` | Project root | Read for project metadata such as `name`, `version`, `engines.node`, `type` (module system), and dependency lists. Used by Node.js preset DataSources and the `loadPackageField()` utility. |
| `composer.json` | Project root | Read by PHP-family presets (CakePHP 2, Laravel, Symfony) to extract `require`, `require-dev` dependencies, and PHP version constraints. |
| `.env.example` / `.env` | Project root | Parsed by Laravel and Symfony presets to extract environment variable keys and default values for documentation. |
| `analysis.json` | `.sdd-forge/output/analysis.json` | Generated by `sdd-forge scan`. Contains the enriched source code analysis used by downstream commands (`data`, `text`, `readme`). |

### Configuration Reference

<!-- {{text[mode=deep]: Describe all configuration fields in table format. Include field name, required/optional, type, default value, and description. Extract from validation logic and default value definitions in the source code.}} -->

The table below lists all fields recognized in `.sdd-forge/config.json`. Validation is performed by `validateConfig()` in `src/lib/types.js` at load time; invalid values cause an error with a combined list of all violations.

| Field | Required | Type | Default | Description |
|---|---|---|---|---|
| `lang` | **Required** | `string` | `"en"` | Operating language for the CLI interface, AGENTS.md templates, skills, and spec output. Commonly `"en"` or `"ja"`. |
| `type` | **Required** | `string` | ‚Äî | Project type identifier. Determines which preset chain is loaded. Examples: `"webapp/cakephp2"`, `"webapp/laravel"`, `"webapp/symfony"`, `"cli/node-cli"`, `"library"`. Short aliases (e.g. `"cakephp2"`) are resolved automatically via `TYPE_ALIASES`. |
| `docs` | **Required** | `object` | ‚Äî | Documentation output configuration (see sub-fields below). |
| `docs.languages` | **Required** | `string[]` | ‚Äî | Non-empty array of output language codes. Example: `["en"]`, `["ja", "en"]`. |
| `docs.defaultLanguage` | **Required** | `string` | ‚Äî | Must be one of the values in `docs.languages`. The primary language in which documentation is authored. |
| `docs.mode` | Optional | `string` | `"translate"` | How non-default languages are produced. Accepted values: `"translate"`, `"generate"`. |
| `docs.style` | Optional | `object` | ‚Äî | Document style settings (see sub-fields below). |
| `docs.style.purpose` | Required if `docs.style` is set | `string` | ‚Äî | Document purpose. Preset choices: `"developer-guide"`, `"user-guide"`, `"api-reference"`, or any custom string. |
| `docs.style.tone` | Required if `docs.style` is set | `string` | ‚Äî | Writing tone. Must be one of: `"polite"`, `"formal"`, `"casual"`. |
| `docs.style.customInstruction` | Optional | `string` | ‚Äî | Free-form additional instruction appended to AI prompts for text generation. |
| `docs.enrichBatchSize` | Optional | `number` | ‚Äî | Number of analysis entries to process per AI batch during the `enrich` step. |
| `docs.enrichBatchLines` | Optional | `number` | ‚Äî | Maximum number of source lines per enrich batch. |
| `concurrency` | Optional | `number` | `5` | Maximum number of files processed in parallel during scan, data, and text commands. Must be a positive number. |
| `chapters` | Optional | `string[]` | (from preset) | Ordered list of chapter filenames. Overrides the chapter order defined in the preset's `preset.json`. Example: `["overview.md", "architecture.md", "api.md"]`. |
| `scan` | Optional | `object` | (from preset) | Scan target configuration. |
| `scan.include` | Required if `scan` is set | `string[]` | ‚Äî | Non-empty array of glob patterns for files to include in source scanning. Example: `["src/**/*.js", "lib/**/*.ts"]`. |
| `scan.exclude` | Optional | `string[]` | `[]` | Array of glob patterns to exclude from scanning. |
| `agent` | Optional | `object` | ‚Äî | AI agent invocation settings. |
| `agent.default` | Optional | `string` | ‚Äî | Name of the default agent provider (must match a key in `agent.providers`). Example: `"claude"`. |
| `agent.workDir` | Optional | `string` | `".tmp"` | Working directory for agent execution, resolved relative to project root. |
| `agent.timeout` | Optional | `number` | `300` | Agent execution timeout in seconds. Must be a positive number. |
| `agent.providers` | Optional | `object` | ‚Äî | Map of provider name ‚Üí provider definition. Each provider must have `command` (non-empty string) and `args` (array). |
| `agent.providers.<name>.command` | Required per provider | `string` | ‚Äî | Executable command name. Example: `"claude"`, `"codex"`. |
| `agent.providers.<name>.args` | Required per provider | `string[]` | ‚Äî | Command arguments. Use `{{PROMPT}}` as a placeholder for the prompt text. |
| `agent.providers.<name>.systemPromptFlag` | Optional | `string` | ‚Äî | CLI flag for passing system prompts. Example: `"--system-prompt"`. |
| `agent.providers.<name>.profiles` | Optional | `object` | ‚Äî | Named argument profiles. Each profile is an array of extra args prepended to the base args. Example: `{ "opus": ["--model", "opus"] }`. |
| `agent.commands` | Optional | `object` | ‚Äî | Per-command agent and profile overrides. Keys are dot-separated command IDs. Lookup order: exact match ‚Üí parent fallback (e.g. `"docs.review"` ‚Üí `"docs"`). |
| `agent.commands.<id>.agent` | Optional | `string` | (from `agent.default`) | Provider name override for this command. |
| `agent.commands.<id>.profile` | Optional | `string` | `"default"` | Profile name to use for this command. |
| `flow` | Optional | `object` | ‚Äî | SDD flow configuration. |
| `flow.merge` | Optional | `string` | `"squash"` | Git merge strategy for the flow merge step. Must be one of: `"squash"`, `"ff-only"`, `"merge"`. |
| `projects` | Optional | `object` | ‚Äî | Monorepo sub-project definitions. Keys are relative paths, values are objects with a `type` field. Example: `{ "apps/web": { "type": "nextjs" } }`. |

### Customization Points

<!-- {{text[mode=deep]: Describe items that users can customize. Extract configurable items from the source code and include configuration examples for each.}} -->

**Project Type and Preset Selection**

The `type` field determines which preset chain is loaded, controlling scan patterns, chapter structure, and available DataSources. sdd-forge resolves type aliases automatically, so you can use short names.

```json
{
  "type": "laravel"
}
```

This resolves to `"webapp/laravel"`, which loads the chain: `base` ‚Üí `webapp` ‚Üí `laravel` (with the `php` language layer).

Available architectures and their presets:

- **webapp**: `cakephp2`, `laravel`, `symfony` (and generic `webapp`)
- **cli**: `node-cli` (and generic `cli`)
- **library**: generic `library`

**Chapter Ordering**

Each preset defines a default chapter list in its `preset.json`. You can override this order in `config.json` to add, remove, or reorder chapters.

```json
{
  "chapters": [
    "overview.md",
    "architecture.md",
    "configuration.md",
    "api.md",
    "development.md"
  ]
}
```

**Document Style**

Control the tone and purpose of generated documentation through `docs.style`.

```json
{
  "docs": {
    "languages": ["en", "ja"],
    "defaultLanguage": "en",
    "mode": "translate",
    "style": {
      "purpose": "user-guide",
      "tone": "casual",
      "customInstruction": "Use short sentences. Avoid jargon where possible."
    }
  }
}
```

**AI Agent Providers and Profiles**

You can register multiple AI providers and switch between them on a per-command basis. Profiles let you vary model selection without duplicating provider definitions.

```json
{
  "agent": {
    "default": "claude",
    "workDir": ".tmp",
    "timeout": 600,
    "providers": {
      "claude": {
        "command": "claude",
        "args": ["-p", "{{PROMPT}}"],
        "systemPromptFlag": "--system-prompt",
        "profiles": {
          "default": ["--model", "sonnet"],
          "opus": ["--model", "opus"]
        }
      }
    },
    "commands": {
      "docs.text": { "agent": "claude", "profile": "default" },
      "docs.forge": { "agent": "claude", "profile": "opus" },
      "spec.gate": { "agent": "claude", "profile": "opus" }
    }
  }
}
```

In this example, `docs.forge` and `spec.gate` use the `opus` profile (prepending `["--model", "opus"]` to the base args), while other documentation commands use the `sonnet` model via the `default` profile.

**Scan Targets**

For presets that do not provide scan patterns, or to override preset defaults, configure `scan` directly.

```json
{
  "scan": {
    "include": ["src/**/*.ts", "lib/**/*.js"],
    "exclude": ["**/*.test.ts", "**/*.spec.js"]
  }
}
```

**Concurrency**

Adjust the number of files processed in parallel. Higher values speed up processing on machines with more resources, but may increase memory usage.

```json
{
  "concurrency": 10
}
```

The default is `5`.

### Environment Variables

<!-- {{text[mode=deep]: List all environment variables referenced by the tool and their purposes in table format. Extract from process.env references in the source code.}} -->

sdd-forge reads the following environment variables at runtime. All are optional; the tool operates without them in typical usage.

| Variable | Purpose | Default Behavior |
|---|---|---|
| `SDD_WORK_ROOT` | Overrides the work root directory (where `.sdd-forge/`, `docs/`, and `specs/` reside). When set, `repoRoot()` returns this value instead of detecting the Git repository root. Useful in project mode or CI environments where the work directory differs from the source. | Detected via `git rev-parse --show-toplevel`, falling back to `process.cwd()`. |
| `SDD_SOURCE_ROOT` | Overrides the source root directory (where the actual source code lives). Used in project mode to separate the source tree from the work tree. | Falls back to the same value as `repoRoot()`. |
| `CLAUDECODE` | Not read by sdd-forge itself, but explicitly **deleted** from the environment before spawning AI agent child processes. This prevents the Claude CLI's internal `CLAUDECODE` variable from being inherited, which could interfere with agent execution. | Removed from the child process environment in `buildAgentInvocation()`. |

To use separate source and work directories (e.g. when the documentation output should live outside the source repository), set both variables before running commands:

```bash
export SDD_SOURCE_ROOT=/path/to/source
export SDD_WORK_ROOT=/path/to/workdir
sdd-forge docs build
```

# Configuration and Customization

## Description

<!-- {{text({prompt: "Write a 1-2 sentence overview of this chapter. Include the types of config files, range of configurable items, and customization points."})}} -->

sdd-forge is controlled primarily through a single JSON configuration file per project, supplemented by preset definitions and agent instruction files generated during setup. The configuration covers project type and scanning behavior, documentation output language and style, AI agent provider selection, SDD flow strategy, and several optional experimental features.

### Configuration Files

| File | Location | Role |
|---|---|---|
| `config.json` | `.sdd-forge/config.json` | Primary configuration file read at runtime. Defines project type, operating language, documentation settings, agent providers, flow behavior, and all other configurable options. |
| `preset.json` | `src/presets/<name>/preset.json` | Bundled preset definition files (shipped with the package). Each preset declares file scan patterns, chapter structure, and framework-specific defaults. Users select a preset via the `type` field in `config.json`. |
| `AGENTS.md` / `CLAUDE.md` | Project root | Agent instruction file generated by `sdd-forge agents` during setup. `CLAUDE.md` is a symbolic link to `AGENTS.md`. Not read as configuration, but consulted by AI agents at runtime. |
| `package.json` | Project root | Read selectively via `loadPackageField()` to extract project metadata such as the package name used during documentation generation. |

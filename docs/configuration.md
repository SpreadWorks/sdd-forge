# 03. Configuration and Customization

## Description

<!-- {{text: Write a 1-2 sentence overview of this chapter. Include the types of config files, range of configurable items, and customization points.}} -->
sdd-forge is configured primarily through `.sdd-forge/config.json` and `preset.json` files. These files control project type, output languages, document style, AI agent providers, scan targets, concurrency limits, and SDD flow behavior, providing extensive customization over every stage of the documentation generation pipeline.
<!-- {{/text}} -->

## Content

### Configuration Files

<!-- {{text: List all configuration files this tool reads, including their locations and roles, in table format. Extract from file reading logic in the source code.}} -->
The following table lists the configuration files that sdd-forge reads during operation.

| File | Location | Role |
|---|---|---|
| `config.json` | `.sdd-forge/config.json` | Primary project configuration. Holds output language, project type, document style, agent providers, scan rules, concurrency limits, and flow settings. Generated by `sdd-forge setup`. |
| `preset.json` | `src/presets/{key}/preset.json` | Preset manifest defining architecture type, label, aliases, scan patterns, and chapter ordering for each supported framework or project type. Discovered automatically at startup. |
| `package.json` | Project root | Read via `loadPackageField()` to extract project metadata such as name and version. |
| `analysis.json` | `.sdd-forge/output/analysis.json` | Generated output from `sdd-forge scan`. Stores the enriched source code analysis that downstream commands (`data`, `text`, etc.) consume. |

### Configuration Reference

<!-- {{text[mode=deep]: Describe all configuration fields in table format. Include field name, required/optional, type, default value, and description. Extract from validation logic and default value definitions in the source code.}} -->
All fields are defined in `.sdd-forge/config.json`. The `validateConfig()` function in `src/lib/types.js` enforces the schema below.

| Field | Required | Type | Default | Description |
|---|---|---|---|---|
| `output` | Yes | `Object` | ‚Äî | Output language configuration. |
| `output.languages` | Yes | `string[]` | ‚Äî | Non-empty array of output language codes (e.g. `["en"]`, `["en", "ja"]`). |
| `output.default` | Yes | `string` | ‚Äî | Default output language. Must be one of `output.languages`. |
| `output.mode` | No | `string` | `"translate"` | How non-default languages are produced. Accepts `"translate"` or `"generate"`. |
| `lang` | Yes | `string` | ‚Äî | Operating language for CLI messages, AGENTS.md, skills, and specs. Falls back to `"en"` when config is unavailable. |
| `type` | Yes | `string` | ‚Äî | Project type identifier. Accepts a canonical path (e.g. `"webapp/cakephp2"`) or a short alias (e.g. `"laravel"`), which is resolved via `TYPE_ALIASES`. |
| `limits` | No | `Object` | ‚Äî | Limit settings for agent and concurrency control. |
| `limits.agentTimeout` | No | `number` | ‚Äî | Agent execution timeout in seconds. |
| `limits.concurrency` | No | `number` | `5` | Maximum number of files processed in parallel. Resolved by `resolveConcurrency()`. |
| `documentStyle` | No | `Object` | ‚Äî | Controls the style of generated documentation. |
| `documentStyle.purpose` | Cond. | `string` | ‚Äî | Required when `documentStyle` is present. Accepts `"developer-guide"`, `"user-guide"`, `"api-reference"`, or a custom string. |
| `documentStyle.tone` | Cond. | `string` | ‚Äî | Required when `documentStyle` is present. Must be `"polite"`, `"formal"`, or `"casual"`. |
| `documentStyle.customInstruction` | No | `string` | ‚Äî | Free-form additional instructions passed to the AI when generating text. |
| `textFill` | No | `Object` | ‚Äî | Settings for the `text` command's output processing. |
| `textFill.projectContext` | No | `string` | ‚Äî | Project overview text provided as context to the AI. |
| `textFill.preamblePatterns` | No | `Object[]` | ‚Äî | Array of `{ pattern, flags }` objects. Regex patterns matched against LLM output prefixes that should be stripped from generated text. |
| `scan` | No | `Object` | ‚Äî | Source file scanning configuration. |
| `scan.include` | Cond. | `string[]` | ‚Äî | Required when `scan` is present. Non-empty array of glob patterns for files to include. |
| `scan.exclude` | No | `string[]` | ‚Äî | Glob patterns for files to exclude from scanning. |
| `chapters` | No | `string[]` | ‚Äî | Ordered list of chapter file names (e.g. `["overview", "architecture"]`). Overrides the chapter order defined in the preset. |
| `agentWorkDir` | No | `string` | `".tmp"` | Working directory for agent file operations. |
| `flow` | No | `Object` | ‚Äî | SDD flow configuration. |
| `flow.merge` | No | `string` | `"squash"` | Merge strategy for flow completion. Accepts `"squash"`, `"ff-only"`, or `"merge"`. |
| `providers` | No | `Object` | ‚Äî | Named AI agent provider definitions. Each key is a provider name. |
| `providers.<name>.command` | Cond. | `string` | ‚Äî | Required for each provider. The executable command (e.g. `"claude"`, `"codex"`). |
| `providers.<name>.args` | Cond. | `string[]` | ‚Äî | Required for each provider. Command arguments. Use `{{PROMPT}}` as a placeholder for the prompt text. |
| `providers.<name>.name` | No | `string` | ‚Äî | Display name for the provider. |
| `providers.<name>.timeoutMs` | No | `number` | ‚Äî | Provider-specific timeout in milliseconds. |
| `providers.<name>.systemPromptFlag` | No | `string` | ‚Äî | CLI flag name for passing a system prompt (e.g. `"--system-prompt"`). |

### Customization Points

<!-- {{text[mode=deep]: Describe items that users can customize. Extract configurable items from the source code and include configuration examples for each.}} -->
Users can tailor sdd-forge behavior through several configuration areas.

**Project Type Selection**

The `type` field determines which preset is loaded, controlling scan patterns, chapter structure, and available DataSources. Short aliases are supported and resolved automatically.

```json
{ "type": "webapp/laravel" }
```

or equivalently:

```json
{ "type": "laravel" }
```

**Document Style**

The `documentStyle` object controls the purpose and tone of generated prose. Adding a `customInstruction` lets you inject project-specific writing guidelines.

```json
{
  "documentStyle": {
    "purpose": "user-guide",
    "tone": "casual",
    "customInstruction": "Use concrete examples wherever possible."
  }
}
```

**Multi-Language Output**

Configure `output` to generate documentation in multiple languages. The `mode` field controls whether secondary languages are machine-translated from the default or generated independently.

```json
{
  "output": {
    "languages": ["en", "ja"],
    "default": "en",
    "mode": "translate"
  }
}
```

**Chapter Ordering**

Override the preset's default chapter sequence by listing chapter file names in the desired order.

```json
{
  "chapters": ["overview", "architecture", "configuration", "cli_commands"]
}
```

**AI Agent Providers**

Define one or more agent providers with their commands, arguments, and optional profiles. The `{{PROMPT}}` placeholder is replaced at runtime with the actual prompt text.

```json
{
  "providers": {
    "claude": {
      "name": "claude-cli",
      "command": "claude",
      "args": ["-p", "{{PROMPT}}"],
      "systemPromptFlag": "--system-prompt"
    },
    "codex": {
      "name": "codex-cli",
      "command": "codex",
      "args": ["exec", "--full-auto", "-C", ".tmp", "{{PROMPT}}"]
    }
  }
}
```

**Concurrency and Timeout Limits**

Adjust parallel processing and agent timeout thresholds to match your hardware or CI environment.

```json
{
  "limits": {
    "concurrency": 10,
    "agentTimeout": 600
  }
}
```

**Preamble Stripping**

LLM responses sometimes include unwanted introductory phrases. The `preamblePatterns` setting removes them automatically.

```json
{
  "textFill": {
    "preamblePatterns": [
      { "pattern": "^(Here is|‰ª•‰∏ã„Å´|Based on)", "flags": "i" }
    ]
  }
}
```

**Flow Merge Strategy**

Control how SDD flow branches are merged back into the main branch.

```json
{
  "flow": {
    "merge": "ff-only"
  }
}
```

### Environment Variables

<!-- {{text[mode=deep]: List all environment variables referenced by the tool and their purposes in table format. Extract from process.env references in the source code.}} -->
sdd-forge reads the following environment variables at runtime. All are optional.

| Variable | Purpose | Default Behavior |
|---|---|---|
| `SDD_WORK_ROOT` | Overrides the work root directory where sdd-forge looks for `.sdd-forge/`, `docs/`, and `specs/`. Used in project mode when the source code and output directories differ. | Detected via `git rev-parse --show-toplevel`. Falls back to the current working directory if not inside a Git repository. |
| `SDD_SOURCE_ROOT` | Overrides the source code root directory used for file scanning and analysis. Allows the scan target to be separate from the work root. | Same as the work root (the value returned by `repoRoot()`). |
| `CLAUDECODE` | Not read by sdd-forge itself. This variable is explicitly deleted from the environment before spawning AI agent child processes, preventing the Claude CLI's internal marker from being inherited by nested invocations. | N/A ‚Äî removed from the child process environment in `src/lib/agent.js`. |

These variables are typically set by wrapper scripts or CI pipelines rather than by end users directly. For most single-repository projects, neither `SDD_WORK_ROOT` nor `SDD_SOURCE_ROOT` needs to be set because sdd-forge auto-detects the repository root.

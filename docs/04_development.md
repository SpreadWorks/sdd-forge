# 04. 開発ガイド

## 説明

<!-- @text: この章の概要を1〜2文で記述してください。開発環境セットアップ、テスト構成を踏まえること。 -->

この章では、sdd-forge のローカル開発環境のセットアップ手順とテスト実行方法を説明します。Node.js 組み込みのテストランナーを活用したテスト構成についても併せて記述しています。

## 内容

### 環境セットアップ

<!-- @text: ローカル開発環境のセットアップ手順を記述してください。 -->

sdd-forge の開発には **Node.js 18.0.0 以上**が必要です。外部ライブラリへの依存はなく、Node.js の組み込みモジュールのみで動作します。

1. リポジトリをクローンします。
2. `node --version` で Node.js のバージョンが 18.0.0 以上であることを確認します。
3. `npm install` を実行します（外部依存はありませんが、開発環境の準備として実行してください）。
4. `npm link` を実行すると、ローカルビルドの `sdd-forge` コマンドをグローバルで利用できます。

### ローカル開発手順

<!-- @text: ローカル開発の手順（起動→コーディング→テスト→確認）を記述してください。 -->

1. `src/` 以下のソースファイルを編集します。パッケージは ES モジュール（`"type": "module"`）で記述します。
2. `npm run test` を実行してテストが通ることを確認します。
3. `sdd-forge <subcommand>` を実行して動作を確認します（`npm link` 済みの場合）。
4. ドキュメントに変更が必要な場合は、`sdd-forge forge --prompt "<変更内容の要約>"` を実行して docs を更新します。
5. `sdd-forge review` で品質チェックを行い、PASS するまで forge を繰り返します。

### SDD ツール

| コマンド | 説明 |
| --- | --- |
| `sdd-forge spec --title "..."` | spec 初期化（feature ブランチ + spec.md 作成） |
| `sdd-forge gate --spec ...` | spec ゲート（未解決事項チェック） |
| `sdd-forge init` | docs 初期化（テンプレートから docs/ を生成） |
| `sdd-forge review` | docs レビュー（構造・内容・網羅性チェック） |
| `sdd-forge forge --prompt "..."` | docs 反復改善 |
| `sdd-forge flow --request "..."` | SDD フロー自動実行 |

### テスト構成

<!-- @text: テストフレームワークとテスト実行方法を説明してください。 -->

テストには Node.js 18 以上で利用可能な組み込みテストランナー（`node --test`）を使用しています。外部のテストフレームワークへの依存はありません。`npm run test` を実行すると、`tests/` 以下の `*.test.js` ファイルが自動検出され、順次実行されます。

```
# 内部的に次のコマンドを実行します:
# find tests -name '*.test.js' | xargs node --test
```

テストファイルは `find` で自動検出されるため、新しいテストファイルを `tests/` 以下に追加するだけで実行対象に含まれます。

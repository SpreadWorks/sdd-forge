# 04. 開発ガイド

## 説明

<!-- @text: この章の概要を1〜2文で記述してください。開発環境セットアップ、テスト構成を踏まえること。 -->

この章では、sdd-forge のローカル開発環境のセットアップ手順と、Node.js 組み込みテストランナーを使ったテスト実行方法を説明します。機能追加・修正を行う際は SDD フローに従って開発を進めてください。

## 内容

### 環境セットアップ

<!-- @text: ローカル開発環境のセットアップ手順を記述してください。 -->

**前提条件:** Node.js 18.0.0 以上がインストールされていること。

```bash
# リポジトリをクローン
git clone <repository-url>
cd sdd-forge

# 依存パッケージのインストール（外部依存なし、Node.js 組み込みのみ使用）
npm install

# グローバルにリンクしてローカル CLI として利用可能にする
npm link
```

`npm link` 後は `sdd-forge` コマンドがローカルのソースを直接参照するため、コードを編集すると即座に反映されます。

### ローカル開発手順

<!-- @text: ローカル開発の手順（起動→コーディング→テスト→確認）を記述してください。 -->

1. **ブランチ作成**: 機能追加・修正を開始する前に SDD フローに従って spec を作成します。
   ```bash
   sdd-forge spec --title "機能名"
   ```
2. **コーディング**: `src/` 以下の該当ファイルを編集します。ES モジュール形式（`import`/`export`）で記述してください。
3. **テスト実行**: 編集後は必ずテストを実行し、既存機能への影響がないことを確認します。
   ```bash
   npm test
   ```
4. **動作確認**: `npm link` 済みの `sdd-forge` コマンドを使って実際の CLI 動作を確認します。
5. **docs 更新**: 実装完了後は `sdd-forge forge` で docs を更新し、`sdd-forge review` で品質チェックを行います。

### SDD ツール

| コマンド | 説明 |
| --- | --- |
| `sdd-forge spec --title "..."` | spec 初期化（feature ブランチ + spec.md 作成） |
| `sdd-forge gate --spec ...` | spec ゲート（未解決事項チェック） |
| `sdd-forge init` | docs 初期化（テンプレートから docs/ を生成） |
| `sdd-forge review` | docs レビュー（構造・内容・網羅性チェック） |
| `sdd-forge forge --prompt "..."` | docs 反復改善 |
| `sdd-forge flow --request "..."` | SDD フロー自動実行 |

### テスト構成

<!-- @text: テストフレームワークとテスト実行方法を説明してください。 -->

テストには Node.js 組み込みのテストランナー（`node --test`）を使用しています。外部テストフレームワークへの依存はありません。

テストの実行コマンドは以下のとおりです。

```bash
npm test
```

# 内部的に次のコマンドを実行します:
# find tests -name '*.test.js' | xargs node --test
```

テストファイルは `find` で自動検出されるため、新しいテストファイルを `tests/` 以下に追加するだけで実行対象に含まれます。

# 04. 開発ガイド

## 説明

<!-- @text: この章の概要を1〜2文で記述してください。開発環境セットアップ、テスト構成を踏まえること。 -->

本章では、sdd-forge の開発環境セットアップからローカル開発の手順、テスト実行までを説明します。外部依存パッケージがないため `npm install` と `npm link` のみで開発環境を構築でき、テストは Node.js 標準の `node:test` モジュールと `node --test` コマンドで実行します。


## 内容

### 環境セットアップ

<!-- @text: ローカル開発環境のセットアップ手順を記述してください。 -->

リポジトリをクローン後、プロジェクトルートで以下を実行してください。

```bash
npm install
npm link
```

`npm link` によりグローバルに `sdd-forge` コマンドが登録され、`src/` 配下のソースを直接参照した状態でコマンドを利用できます。外部依存パッケージはないため、追加のインストール作業は不要です。動作には Node.js（ESM サポートが必要なため v18 以上を推奨）が必要です。


### ローカル開発手順

<!-- @text: ローカル開発の手順（起動→コーディング→テスト→確認）を記述してください。 -->

開発中は `npm link` を実行してグローバルに `sdd-forge` コマンドをリンクしておくと、ソースを直接参照した状態で動作確認できます。

コーディングは `src/` 配下のファイルを直接編集します。変更はリンク経由で即時反映されるため、再インストールは不要です。

テストは `node --test` コマンドで実行します。テストファイルは `node:test` モジュールを使用して作成してください。

動作確認は実際の対象プロジェクトディレクトリで各サブコマンドを実行することで行います。`sdd-forge help` でコマンド一覧を確認しながら、変更の影響範囲を検証してください。


### SDD ツール

| コマンド | 説明 |
| --- | --- |
| `sdd-forge spec --title "..."` | spec 初期化（feature ブランチ + spec.md 作成） |
| `sdd-forge gate --spec ...` | spec ゲート（未解決事項チェック） |
| `sdd-forge init` | docs 初期化（テンプレートから docs/ を生成） |
| `sdd-forge review` | docs レビュー（構造・内容・網羅性チェック） |
| `sdd-forge forge --prompt "..."` | docs 反復改善 |
| `sdd-forge flow --request "..."` | SDD フロー自動実行 |

### テスト構成

<!-- @text: テストフレームワークとテスト実行方法を説明してください。 -->

現在のプロジェクト構成では、`package.json` に `dependencies` および `devDependencies` は定義されておらず、外部のテストフレームワークは導入されていません。

テストを追加する場合は、Node.js 組み込みの `node:test` モジュールを利用する方法が選択肢の一つです。`node:test` は外部パッケージなしで利用でき、`node --test` コマンドでテストを実行できます。

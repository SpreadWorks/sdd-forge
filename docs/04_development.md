# 04. 開発ガイド

## 説明

<!-- @text: この章の概要を1〜2文で記述してください。開発環境セットアップ、テスト構成を踏まえること。 -->

本章では、sdd-forge の開発環境セットアップ手順とローカル開発フローを説明します。テストは Node.js 組み込みのテストランナーを使用しており、外部フレームワークなしで実行できます。


## 内容

### 環境セットアップ

<!-- @text: ローカル開発環境のセットアップ手順を記述してください。 -->

1. **Node.js 18 以上**をインストールしてください。`node --version` でバージョンを確認できます。
2. リポジトリをクローンし、プロジェクトルートに移動してください。
3. `npm install` を実行して依存関係をインストールします（本パッケージは外部依存がないため、主に開発用パッケージのみがインストールされます）。
4. ローカルで `sdd-forge` コマンドを使用するには、`npm link` を実行してグローバルにリンクしてください。
5. `sdd-forge help` を実行し、コマンド一覧が表示されれば環境セットアップは完了です。


### ローカル開発手順

<!-- @text: ローカル開発の手順（起動→コーディング→テスト→確認）を記述してください。 -->

1. **機能追加・修正の開始**: 機能追加や改修を行う場合は、必ず SDD フローに従ってください。`sdd-forge spec --title "<機能名>"` で spec を作成し、仕様を固めてから実装に入ります。
2. **コーディング**: `src/` 以下のソースファイルを編集します。本プロジェクトは ES modules (`"type": "module"`) を採用しているため、`import`/`export` 構文を使用してください。
3. **テスト実行**: `npm run test` でテストスイート全体を実行し、既存機能に影響がないことを確認してください。
4. **docs の更新**: 実装完了後は `sdd-forge forge --prompt "<変更内容の要約>"` を実行して docs を更新し、`sdd-forge review` で品質チェックを通してください。
5. **コミット**: すべてのチェックが通った後、feature ブランチでコミットします。コミットメッセージは英語で記述してください。


### SDD ツール

| コマンド | 説明 |
| --- | --- |
| `sdd-forge spec --title "..."` | spec 初期化（feature ブランチ + spec.md 作成） |
| `sdd-forge gate --spec ...` | spec ゲート（未解決事項チェック） |
| `sdd-forge init` | docs 初期化（テンプレートから docs/ を生成） |
| `sdd-forge review` | docs レビュー（構造・内容・網羅性チェック） |
| `sdd-forge forge --prompt "..."` | docs 反復改善 |
| `sdd-forge flow --request "..."` | SDD フロー自動実行 |

### テスト構成

<!-- @text: テストフレームワークとテスト実行方法を説明してください。 -->

Node.js 18 以上に標準搭載されているテストランナー（`node --test`）を使用しています。外部のテストフレームワークへの依存はありません。テストの実行は以下のコマンドで行います。

```bash
npm run test
```

# 内部的に次のコマンドを実行します:
# find tests -name '*.test.js' | xargs node --test
```

テストファイルは `find` で自動検出されるため、新しいテストファイルを `tests/` 以下に追加するだけで実行対象に含まれます。

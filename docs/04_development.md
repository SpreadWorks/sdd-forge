# 04. 開発ガイド

## 説明

<!-- @text: この章の概要を1〜2文で記述してください。開発環境セットアップ、テスト構成を踏まえること。 -->

本章では、sdd-forge のローカル開発環境の構築手順とコーディングから動作確認までの開発フローを説明します。テストは Node.js 組み込みのテストランナーを使用しており、専用フレームワークの追加インストールは不要です。

## 内容

### 環境セットアップ

<!-- @text: ローカル開発環境のセットアップ手順を記述してください。 -->

Node.js 18.0.0 以上が必要です。外部依存パッケージは存在せず、Node.js 組み込みモジュールのみを使用しているため、`npm install` 後すぐに開発を開始できます。`npm link` でグローバルシンボリックリンクを作成しておくと、ローカルのソースコードを直接 `sdd-forge` コマンドとして実行できるようになります。

```bash
# リポジトリをクローン
git clone <repository-url>
cd sdd-forge

# 依存パッケージのインストール（外部依存なし、Node.js 組み込みのみ使用）
npm install

# グローバルにリンクしてローカル CLI として利用可能にする
npm link
```

`npm link` 後は `sdd-forge` コマンドがローカルのソースを直接参照するため、コードを編集すると即座に反映されます。

### ローカル開発手順

<!-- @text: ローカル開発の手順（起動→コーディング→テスト→確認）を記述してください。 -->

`npm link` によってローカルソースと CLI が紐付けられた状態で開発を進めます。コードを編集した後はファイルの保存のみで変更が即座に反映されるため、再ビルドは不要です。変更後は `npm test` でテストスイートを実行し、既存の動作が壊れていないことを確認してください。CLI の動作確認は `sdd-forge <subcommand>` を直接実行するか、テスト対象プロジェクト上で `sdd-forge build` などを実行することで行えます。機能追加・修正を行う際は、CLAUDE.md に記載の SDD フローに従い、`sdd-forge spec` でスペックを作成してから実装を開始してください。

### SDD ツール

| コマンド | 説明 |
| --- | --- |
| `sdd-forge spec --title "..."` | spec 初期化（feature ブランチ + spec.md 作成） |
| `sdd-forge gate --spec ...` | spec ゲート（未解決事項チェック） |
| `sdd-forge init` | docs 初期化（テンプレートから docs/ を生成） |
| `sdd-forge review` | docs レビュー（構造・内容・網羅性チェック） |
| `sdd-forge forge --prompt "..."` | docs 反復改善 |
| `sdd-forge flow --request "..."` | SDD フロー自動実行 |

### テスト構成

<!-- @text: テストフレームワークとテスト実行方法を説明してください。 -->

Node.js 18 以上に組み込まれている `node:test` モジュールをテストフレームワークとして使用しています。外部テストライブラリへの依存はありません。テストの実行は `npm test` コマンドで行います。

```bash
npm test
# 内部的に次のコマンドを実行します:
# find tests -name '*.test.js' | xargs node --test
```

テストファイルは `find` で自動検出されるため、新しいテストファイルを `tests/` 以下に追加するだけで実行対象に含まれます。
